## Goal


Auto-instrument LangChain.js agents using official `BaseCallbackHandler` API, producing higher-confidence traces than log parsing alone.


---


## User Story


> As a developer using LangChain for agent workflows, I want to add one callback and see a complete execution tree with explicit parent-child relationships, without wrapping every function manually.


---


## Scope


### In Scope
- `BaseCallbackHandler` implementation (official API)
- LLM start/end callbacks → LLM spans
- Tool start/end callbacks → TOOL spans
- Chain start/end callbacks → CHAIN spans
- Agent action/end callbacks → AGENT spans
- AsyncLocalStorage for parent tracking
- Explicit parentId relationships
- Token usage metadata (input/output tokens)


### Out of Scope
- Monkey-patching LangChain internals
- Auto-import / magic instrumentation
- Other framework adapters (AI SDK, Mastra)
- NestJS module wrapper (useful but later)
- Custom LangChain chain instrumentation
- Prompt/output capture (metadata only for v0.5)
- Cost calculation (token usage yes, cost no)

## API


### Basic Usage


```typescript
import { AgentInspectCallback } from "@agent-inspect/langchain";


const callback = new AgentInspectCallback({
 runName: "support-agent-eval",
 traceDir: "./.agent-runs",
 capture: "metadata-only",  // "none" | "metadata-only" | "preview"
});


await agent.invoke(
 { messages },
 { callbacks: [callback] }
);
```


### With Manual Steps


```typescript
import { step } from "agent-inspect";
import { AgentInspectCallback } from "@agent-inspect/langchain";


await inspectRun("eval-run", async () => {
 // Manual step
 const input = await step("prepare-input", async () => {
   return { query: "book hotel" };
 });


 // LangChain agent with callback
 const result = await agent.invoke(
   input,
   { callbacks: [new AgentInspectCallback()] }
 );


 return result;
});
```

## Callback Contract


### LangChain.js BaseCallbackHandler


Reference: [LangChain Callbacks](https://js.langchain.com/docs/modules/callbacks/)


```typescript
import { BaseCallbackHandler } from "@langchain/core/callbacks/base";


export interface AgentInspectCallbackOptions {
 runName?: string;
 traceDir?: string;
 silent?: boolean;
 capture?: "none" | "metadata-only" | "preview";
 redact?: RedactionRule[];
}


export class AgentInspectCallback extends BaseCallbackHandler {
 constructor(options?: AgentInspectCallbackOptions);


 // LLM callbacks
 async handleLLMStart(llm, prompts, runId, parentRunId?, tags?, metadata?);
 async handleLLMEnd(output, runId, parentRunId?);
 async handleLLMError(err, runId, parentRunId?);


 // Tool callbacks
 async handleToolStart(tool, input, runId, parentRunId?);
 async handleToolEnd(output, runId, parentRunId?);
 async handleToolError(err, runId, parentRunId?);


 // Chain callbacks
 async handleChainStart(chain, inputs, runId, parentRunId?);
 async handleChainEnd(outputs, runId, parentRunId?);
 async handleChainError(err, runId, parentRunId?);


 // Agent callbacks
 async handleAgentAction(action, runId, parentRunId?);
 async handleAgentEnd(action, runId, parentRunId?);
}
```

## Event Mapping


| LangChain Callback | InspectEvent Kind | Notes |
|--------------------|-------------------|-------|
| `handleLLMStart`/`End` | `LLM` | Model name, tokens from metadata |
| `handleToolStart`/`End` | `TOOL` | Tool name, inputs/outputs |
| `handleChainStart`/`End` | `CHAIN` | Chain name, nested structure |
| `handleAgentAction`/`End` | `AGENT` | Agent reasoning, decisions |
| `*Error` callbacks | Set `status: "error"` | Preserve error message |

## Parent Tracking with AsyncLocalStorage


### Challenge


LangChain callbacks provide `runId` and `parentRunId`, but we also need to connect them to manual `step()` calls in the same execution.


### Solution: AsyncLocalStorage


```typescript
import { AsyncLocalStorage } from "node:async_hooks";


const tracingContext = new AsyncLocalStorage<{
 runId: string;
 parentStepId?: string;
}>();


// In AgentInspectCallback
async handleLLMStart(llm, prompts, runId, parentRunId) {
 const context = tracingContext.getStore();
  const event: InspectEvent = {
   eventId: runId,
   runId: context?.runId || runId,  // Use context runId if available
   parentId: parentRunId || context?.parentStepId,  // LangChain or manual parent
   name: llm._llmType(),
   kind: "LLM",
   // ...
 };
  // Store event
 eventStore.add(event);
}
```


### Integration with `inspectRun()`


```typescript
// In packages/core/src/inspect-run.ts
export async function inspectRun<T>(
 name: string,
 fn: () => Promise<T>
): Promise<T> {
 const runId = nanoid();
  return tracingContext.run({ runId }, async () => {
   // Start run event
   // ...
  
   const result = await fn();
  
   // End run event
   // ...
  
   return result;
 });
}
```

## Technical Design


### Architecture


```
┌──────────────────────────────────────┐
│  LangChain Agent                     │
│  - invoke({ callbacks: [callback] }) │
└──────────────────────────────────────┘
             ↓
┌──────────────────────────────────────┐
│  AgentInspectCallback                │
│  - handleLLMStart/End                │
│  - handleToolStart/End               │
│  - handleChainStart/End              │
│  - handleAgentAction/End             │
└──────────────────────────────────────┘
             ↓
┌──────────────────────────────────────┐
│  AsyncLocalStorage Context           │
│  - Track current runId               │
│  - Track parent stepId               │
└──────────────────────────────────────┘
             ↓
┌──────────────────────────────────────┐
│  Event Store (shared with core)      │
│  - Same InspectEvent model           │
│  - Explicit confidence="explicit"    │
└──────────────────────────────────────┘
```


### Key Implementation Points


**1. Token Tracking**
```typescript
async handleLLMEnd(output, runId, parentRunId) {
 const tokenUsage = output.llmOutput?.tokenUsage;
  const event: InspectEvent = {
   // ...
   attributes: {
     model: output.llmOutput?.model,
     tokens: tokenUsage ? {
       input: tokenUsage.promptTokens,
       output: tokenUsage.completionTokens,
       total: tokenUsage.totalTokens,
     } : undefined,
   },
 };
}
```


**2. Prompt/Output Capture**
```typescript
// metadata-only mode (default)
attributes: {
 promptLength: prompts[0].length,
 outputLength: output.generations[0][0].text.length,
}


// preview mode (first 200 chars)
attributes: {
 promptPreview: prompts[0].slice(0, 200),
 outputPreview: output.generations[0][0].text.slice(0, 200),
}


// none mode
attributes: {}
```


**3. Error Handling**
```typescript
async handleLLMError(err, runId, parentRunId) {
 const event: InspectEvent = {
   // ...
   status: "error",
   attributes: {
     errorMessage: err.message,
     errorType: err.constructor.name,
   },
 };
}
```

## Testing Strategy


### Unit Tests
- [ ] Each callback method creates correct InspectEvent
- [ ] Token extraction from LangChain metadata
- [ ] Error callbacks set status="error"
- [ ] Parent tracking with parentRunId


### Integration Tests
- [ ] LangChain agent with callback produces tree
- [ ] Nested chains maintain parent-child relationships
- [ ] Mixed manual steps + LangChain callback in same run
- [ ] Capture modes (none, metadata, preview)


### Stress Tests (CRITICAL)
- [ ] 100 concurrent tool calls with Promise.all
- [ ] AsyncLocalStorage context isolation
- [ ] No parent leakage across concurrent runs
- [ ] Memory leak check with long-running agent


**Stress Test as Release Gate**:
```typescript
test("100 concurrent tools maintain correct parents", async () => {
 const tools = Array.from({ length: 100 }, (_, i) =>
   createMockTool(`tool-${i}`)
 );
  await agent.invoke(
   { tools, query: "test" },
   { callbacks: [new AgentInspectCallback()] }
 );
  const tree = await loadTrace(runId);
  // Verify no parent misattribution
 for (const event of tree.events) {
   if (event.parentId) {
     expect(tree.events.find(e => e.eventId === event.parentId)).toBeDefined();
   }
 }
});
```

## Go/No-Go Gate


**GO IF**:
- LangChain adapter produces significantly better traces than log parsing
- AsyncLocalStorage works reliably (no parent leakage)
- 1+ team adopts for eval debugging
- No major performance issues


**NO-GO IF**:
- Log parsing already sufficient for most cases
- AsyncLocalStorage too fragile
- LangChain API changes too frequently
- Maintenance burden too high


**If NO-GO**: Focus on log parsing quality, skip adapters

## Known Limitations


- Only works with LangChain.js (not Python)
- Requires explicit callback registration
- LangChain internal chains may have incomplete callbacks
- Custom chains need manual instrumentation
