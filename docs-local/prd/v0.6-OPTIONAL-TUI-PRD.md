 
# v0.6 Planning Guide: Optional TUI


**Version**: 0.6.0 
**Theme**: Keyboard-driven interactive tree inspection 
**Duration**: 3-4 weeks 
**Risk Level**: MEDIUM (Ink complexity, terminal compatibility) 
**Status**: Planning guide (detailed implementation after v0.3 validates)


---


## ⚠️ Prerequisites


**DO NOT implement v0.6 until**:
- ✅ v0.3 validated and adopted
- ✅ Simple terminal output proves sufficient for most use cases
- ✅ User requests for richer navigation justify complexity


---


## Goal


Provide optional keyboard-driven inspection for traces when simple terminal output is limiting, while keeping simple CLI as default and fallback.


---


## User Story


> As a developer inspecting complex agent runs with 50+ events, I want to navigate the tree with keyboard shortcuts, expand/collapse nodes, and view metadata panels without scrolling through terminal output.


---


## Scope


### In Scope
- Separate optional package (`@agent-inspect/tui`)
- Ink-based React components
- Keyboard navigation (j/k, space, o, q)
- Collapsible tree nodes
- Metadata panel
- Error navigation
- Help screen
- Non-interactive fallback


### Out of Scope
- Mouse support (keyboard only)
- Themes (default colors only)
- Live updating TUI (replay mode only for v0.6)
- Multi-run panels (single run view)
- Prompt editing
- Replay/fork execution


---


## Package Strategy


**Separate optional package** (user must install explicitly):


```bash
npm install @agent-inspect/tui
```


**NOT an optionalDependency** — keep main package lean:
```json
{
 "dependencies": {
   "chalk": "^5.0.0",
   "commander": "^11.0.0",
   "nanoid": "^5.0.0"
 }
 // No @agent-inspect/tui in optionalDependencies
}
```


**CLI detects if installed**:
```typescript
// If @agent-inspect/tui installed
agent-inspect view run_abc --tui  // Uses TUI


// If not installed
agent-inspect view run_abc --tui  // Error: "TUI requires @agent-inspect/tui. Run: npm install @agent-inspect/tui"
```

## Keyboard Shortcuts


| Key | Action |
|-----|--------|
| `j` / `↓` | Next node |
| `k` / `↑` | Previous node |
| `space` | Expand/collapse node |
| `o` | Toggle details panel |
| `e` | Jump to next error |
| `E` | Jump to previous error |
| `g` | Go to top |
| `G` | Go to bottom |
| `?` | Show help |
| `q` | Quit |

### Core Components


**1. RunView (Root)**
```typescript
import React from "react";
import { Box, useInput, useFocus } from "ink";


interface RunViewProps {
 tree: InspectRunTree;
 onExit: () => void;
}


export function RunView({ tree, onExit }: RunViewProps) {
 const [selectedIndex, setSelectedIndex] = useState(0);
 const [expanded, setExpanded] = useState<Set<string>>(new Set());
 const [showDetails, setShowDetails] = useState(false);
 const [showHelp, setShowHelp] = useState(false);


 useInput((input, key) => {
   if (input === "q") onExit();
   if (input === "j" || key.downArrow) setSelectedIndex(i => i + 1);
   if (input === "k" || key.upArrow) setSelectedIndex(i => i - 1);
   if (input === " ") toggleExpanded(selectedIndex);
   if (input === "o") setShowDetails(!showDetails);
   if (input === "?") setShowHelp(!showHelp);
 });


 return (
   <Box flexDirection="column">
     <Header tree={tree} />
     <TreeView
       nodes={tree.children}
       selectedIndex={selectedIndex}
       expanded={expanded}
     />
     {showDetails && <DetailsPanel node={getSelectedNode()} />}
     {showHelp && <HelpScreen />}
     <StatusBar shortcuts={SHORTCUTS} />
   </Box>
 );
}
```


**2. TreeView**
```typescript
interface TreeViewProps {
 nodes: InspectNode[];
 selectedIndex: number;
 expanded: Set<string>;
}


function TreeView({ nodes, selectedIndex, expanded }: TreeViewProps) {
 const flatNodes = flattenTree(nodes, expanded);
  return (
   <Box flexDirection="column">
     {flatNodes.map((node, i) => (
       <NodeRow
         key={node.event.eventId}
         node={node}
         selected={i === selectedIndex}
         expanded={expanded.has(node.event.eventId)}
       />
     ))}
   </Box>
 );
}
```


**3. NodeRow**
```typescript
interface NodeRowProps {
 node: InspectNode;
 selected: boolean;
 expanded: boolean;
}


function NodeRow({ node, selected, expanded }: NodeRowProps) {
 const indent = "  ".repeat(node.depth);
 const icon = node.children.length > 0 ? (expanded ? "▼" : "▶") : " ";
 const statusIcon = getStatusIcon(node.event.status);
 const duration = node.event.durationMs
   ? formatDuration(node.event.durationMs)
   : "";


 return (
   <Box>
     <Text backgroundColor={selected ? "blue" : undefined}>
       {indent}{icon} {node.event.name} {statusIcon} {duration}
     </Text>
   </Box>
 );
}
```


**4. DetailsPanel**
```typescript
function DetailsPanel({ node }: { node: InspectNode }) {
 return (
   <Box flexDirection="column" borderStyle="single" marginTop={1}>
     <Text bold>Details: {node.event.name}</Text>
     <Text>Status: {node.event.status || "unknown"}</Text>
     {node.event.durationMs && (
       <Text>Duration: {formatDuration(node.event.durationMs)}</Text>
     )}
     {node.event.attributes && (
       <>
         <Text>Input: {JSON.stringify(node.event.attributes.input, null, 2)}</Text>
         <Text>Output: {JSON.stringify(node.event.attributes.output, null, 2)}</Text>
       </>
     )}
   </Box>
 );
}
```

## Fallback Strategy


### Detection Logic


```typescript
function shouldUseTUI(): boolean {
 // Check if TUI package installed
 try {
   require.resolve("@agent-inspect/tui");
 } catch {
   return false;
 }


 // Check if TTY
 if (!process.stdout.isTTY) {
   return false;
 }


 // Check terminal size
 const { columns, rows } = process.stdout;
 if (columns < 80 || rows < 24) {
   return false;
 }


 return true;
}
```


### Graceful Degradation


```typescript
export async function view(runId: string, options: any) {
 const tree = await loadTrace(runId);


 if (options.tui && shouldUseTUI()) {
   const { renderTUI } = await import("@agent-inspect/tui");
   await renderTUI(tree);
 } else {
   if (options.tui) {
     console.warn("TUI not available, using simple output");
   }
   console.log(renderTreeSimple(tree));
 }
}
```

## Performance Considerations


### Virtualization


For large trees (100+ nodes):
- Only render visible nodes
- Use `ink-big-text` for scrollable regions
- Lazy load details panel content


### Update Throttling


```typescript
const [tree, setTree] = useState(initialTree);


// Throttle updates to 30fps
const throttledUpdate = useCallback(
 throttle((newTree) => setTree(newTree), 33),
 []
);
```

## Go/No-Go Gate


**GO IF**:
- Users request richer navigation
- Simple output proves limiting for complex traces
- Ink TUI works reliably


**NO-GO IF**:
- Simple output is sufficient for all use cases
- TUI too buggy or complex
- Terminal compatibility issues


**If NO-GO**: Keep simple output only, skip TUI

## Known Limitations


- Keyboard only (no mouse)
- Single terminal tab (no split view)
- Default colors only (no themes in v0.6)
- Replay mode only (live TUI in v0.7)
- Assumes terminal width ≥80 columns
