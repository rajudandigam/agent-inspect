## Stability Checklist


### 1. API Stability


**Core APIs (FROZEN)**:
```typescript
// These must not change in v1.x
export function inspectRun<T>(name: string, fn: () => Promise<T>): Promise<T>;
export function step<T>(name: string, fn: () => Promise<T>): Promise<T>;
export function observe<T>(target: T): T;


// Namespaced helpers (FROZEN)
step.llm<T>(name: string, fn: () => Promise<T>): Promise<T>;
step.tool<T>(name: string, fn: () => Promise<T>): Promise<T>;
step.chain<T>(name: string, fn: () => Promise<T>): Promise<T>;
```


**CLI Commands (FROZEN)**:
```bash
agent-inspect list [options]
agent-inspect view <runId> [options]
agent-inspect logs <file> [options]
agent-inspect tail [options]
agent-inspect export <runId> --format <format>
agent-inspect diff <runA> <runB> [options]
agent-inspect clean [options]
```


**Public TypeScript Interfaces (FROZEN)**:
```typescript
// Must remain backward compatible
export interface InspectEvent { ... }
export interface InspectRunTree { ... }
export interface InspectNode { ... }
export type AttributionConfidence = ...;
export type InspectKind = ...;
```


**JSONL Schema (VERSIONED)**:
```jsonl
{"schemaVersion":"1.0","event":"step_started",...}
```


Schema version `1.0` is frozen. Future changes require `1.1`, `2.0`, etc.


### 2. Deprecation Policy


**Process**:
1. Deprecate in v1.x with console warning
2. Document replacement in CHANGELOG
3. Maintain for 6 months minimum
4. Remove in v2.0


**Example**:
```typescript
// v1.2: Deprecate
export function oldAPI() {
 console.warn("oldAPI() is deprecated. Use newAPI() instead. Will be removed in v2.0.");
 return newAPI();
}


// v2.0: Remove
// (function removed)
```


### 3. Breaking Change Policy


**What requires major version bump**:
- Removing public API
- Changing function signatures
- Changing JSONL schema (non-additive)
- Removing CLI commands or flags
- Changing default behavior


**What can be minor version**:
- Adding optional parameters
- Adding new CLI flags
- Adding new JSONL fields
- New CLI commands
- Performance improvements


### 4. Support Policy


**v1.0 LTS**:
- Bug fixes: 12 months minimum
- Security patches: 18 months minimum
- Node.js version support: Follow Node.js LTS schedule


**Supported Node versions** (at v1.0 launch in May 2026):
- Node 20 (LTS until Apr 2026)
- Node 22 (LTS until Apr 2027)
- Node 24 (LTS until Apr 2029)


Note: Node 18 reaches EOL April 2025, before v1.0 ships.


Drop support when Node.js version reaches EOL.

## Plugin API


### Adapter Interface


**@experimental in v1.0** (may change in v1.1):
```typescript
/**
* @experimental
* This interface is experimental and may change in v1.1.
* Use at your own risk until marked stable.
*/
export interface AgentInspectAdapter {
 name: string;
 version: string;
 attach(builder: RunTreeBuilder): Promise<() => void> | (() => void);
}


// Usage
import { registerAdapter } from "agent-inspect";


const myAdapter: AgentInspectAdapter = {
 name: "my-framework",
 version: "1.0.0",
 attach(builder) {
   // Subscribe to framework events
   framework.on("event", (data) => {
     builder.addEvent({
       // ...
     });
   });
  
   // Return cleanup function
   return () => {
     framework.off("event");
   };
 },
};


registerAdapter(myAdapter);
```


### Sink Interface


**@experimental in v1.0** (may change in v1.1):
```typescript
/**
* @experimental
* This interface is experimental and may change in v1.1.
* Only LangChain adapter (v0.5) uses this in v1.0.
* Will stabilize after building more real sinks in v1.1+.
*/
export interface AgentInspectSink {
 name: string;
 version: string;
 onSpanStart(span: InspectEvent): void | Promise<void>;
 onSpanEnd(span: InspectEvent): void | Promise<void>;
 flush(): Promise<void>;
}


// Usage
import { registerSink } from "agent-inspect";


const mySink: AgentInspectSink = {
 name: "my-platform",
 version: "1.0.0",
 onSpanStart(span) {
   // Handle span start
 },
 onSpanEnd(span) {
   // Handle span end
 },
 async flush() {
   // Flush buffered data
 },
};


registerSink(mySink);
```


**Note**: These interfaces are marked @experimental because only one adapter (LangChain) and zero production sinks exist in v1.0. They will be stabilized in v1.1+ after validating with more real-world usage.

### Release Steps


Uses established pnpm + Changesets workflow:


```bash
# 1. Create changeset
pnpm changeset
# Select: major
# Describe: "v1.0.0 - Stable Release"


# 2. Version packages
pnpm changeset version
# Updates package.json, CHANGELOG.md


# 3. Build
pnpm build


# 4. Test built artifacts
pnpm pack
pnpm install -g ./agent-inspect-1.0.0.tgz
agent-inspect --version  # Verify


# 5. Commit & push
git add .
git commit -m "chore: release v1.0.0"
git push


# 6. Publish via GitHub Actions
# CI runs on push to main:
# - Runs tests
# - Publishes to npm via `pnpm changeset publish`
# - Creates GitHub release
# - Tags commit


# 7. Announce
# - GitHub Discussions
# - Internal channels
```
