## Goal


Watch agent execution as a live tree while `npm run dev` runs, without opening a dashboard or browser.


---


## User Story


> As a developer running `npm run dev`, I want to pipe logs into agent-inspect and see an updating execution tree in my terminal without stopping my workflow.


---


## Scope


### In Scope
- Stream logs from stdin or file
- Incremental tree updates
- Group by runId in real-time
- Show running/incomplete events
- Ctrl+C stops cleanly (no trace writes by default)
- Fallback to plain output if non-interactive


### Out of Scope
- Ink TUI (simple terminal updates only)
- Keyboard navigation (view mode only)
- Expand/collapse (flat tree only)
- Diff/replay (separate version)
- Split panel multi-run display (basic list is in-scope)


---


## CLI API


```bash
# Pipe from app
npm run dev 2>&1 | npx agent-inspect tail --format log4js


# Tail a file
npx agent-inspect tail --file ./logs/agent.log --format json


# With config
npx agent-inspect tail \
 --file ./logs/agent.log \
 --config agent-inspect.logs.json


# Filter to specific run
npx agent-inspect tail --run-id decisionId=01fe6bf1


# Control update frequency
npx agent-inspect tail --refresh 100  # ms, default 250
```

## Technical Design


### Architecture


```
┌──────────────────────────────────────┐
│  Log Source                          │
│  - stdin pipe                        │
│  - file tail                         │
└──────────────────────────────────────┘
             ↓
┌──────────────────────────────────────┐
│  Stream Reader                       │
│  - Line-by-line reading              │
│  - Backpressure handling             │
└──────────────────────────────────────┘
             ↓
┌──────────────────────────────────────┐
│  Event Buffer (per run)              │
│  - Ring buffer (max 1000 events)     │
│  - Drops oldest if full              │
└──────────────────────────────────────┘
             ↓
┌──────────────────────────────────────┐
│  Incremental Tree Builder            │
│  - Updates tree as events arrive     │
│  - Tracks in-flight events           │
└──────────────────────────────────────┘
             ↓
┌──────────────────────────────────────┐
│  Terminal Renderer                   │
│  - Clear screen + reprint            │
│  - Throttled updates (250ms default) │
│  - Non-interactive fallback          │
└──────────────────────────────────────┘
```


### Key Components


**1. Stream Reader**
```typescript
interface StreamReaderOptions {
 source: "stdin" | "file";
 filePath?: string;
 follow?: boolean;  // tail -f behavior
}


class StreamReader {
 async *read(): AsyncGenerator<string> {
   // Yield lines as they arrive
 }
}
```


**2. Event Buffer**
```typescript
class RunEventBuffer {
 private maxSize = 1000;
 private events: InspectEvent[] = [];
  add(event: InspectEvent): void {
   if (this.events.length >= this.maxSize) {
     this.events.shift();  // Drop oldest
   }
   this.events.push(event);
 }
}
```


**3. Incremental Renderer**
```typescript
class LiveRenderer {
 private lastRender = 0;
 private throttleMs = 200;
  async render(trees: InspectRunTree[]): Promise<void> {
   const now = Date.now();
   if (now - this.lastRender < this.throttleMs) {
     return;  // Skip, too soon
   }
  
   // Clear screen (ANSI escape)
   process.stdout.write("\x1b[2J\x1b[0f");
  
   // Render trees
   for (const tree of trees) {
     console.log(renderTree(tree));
   }
  
   this.lastRender = now;
 }
}
```


**4. Graceful Exit**
```typescript
process.on("SIGINT", async () => {
 console.log("\nStopping... saving traces...");
  // Save partial traces to disk
 for (const [runId, events] of activeRuns) {
   await saveTrace(runId, events);
 }
  console.log(`Saved ${activeRuns.size} partial traces.`);
 process.exit(0);
});
```

## Performance Considerations


### Throttling
- Default: 200ms between renders
- Configurable via `--refresh` flag
- Skip renders if previous hasn't finished


### Backpressure
- Ring buffer per run (max 1000 events)
- Drop oldest events if buffer full
- Show warning: "Dropped N events (buffer full)"


### Memory Management
- Auto-archive completed runs after 1 minute
- Keep max 10 active runs in memory
- Older runs saved to disk, removed from display


---


## Error Handling


### Malformed Lines
- Skip and continue (don't crash)
- Count skipped lines
- Show summary: "Skipped 12 lines"


### File Disappears (tail mode)
- Retry for 5 seconds
- Fall back to stdin if file not recovered
- Show warning


### Non-Interactive Terminal
- Detect TTY
- Fall back to append-only output
- No clear screen, just append trees

## Go/No-Go Gate


**GO IF**:
- Live tail preferred over batch `logs` command
- Doesn't freeze or crash on rapid output
- Useful for at least 1 real workflow


**NO-GO IF**:
- Performance too poor (lag, freeze)
- No value over batch processing
- Too complex to use


**If NO-GO**: Keep batch `logs` command, skip live tail

## Known Limitations
