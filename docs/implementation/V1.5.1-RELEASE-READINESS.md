# v1.5.1 release readiness

**Status:** ready for maintainer Gate C review — not released.
**Published baseline:** `agent-inspect@1.5.0`, `@agent-inspect/langchain@1.5.0`, `@agent-inspect/tui@1.5.0`.
**Train:** v1.5.1 corrective patch.
**Date:** 2026-06-24.

## Scope

v1.5.1 is a corrective patch before any v1.6.0 work. It addresses the seven confirmed v1.5.0 post-release findings:

1. CLI version reporting used a stale hard-coded value.
2. `list`, `stats`, and `search` had v0.1-only metadata paths.
3. `report --redaction-profile share|strict` protected only the tree section.
4. Mixed v0.1/v0.2 files could be reordered during normalization.
5. Persisted `error.name` could be mapped into the v0.1 stack field.
6. Token usage lost `total` in one conversion path and lacked `cached` vocabulary.
7. CLI and release-readiness documentation contained stale v1.5.0 wording.

## Security notes

- Whole-report redaction is covered by Chunk 2 tests for Markdown and HTML.
- Redaction profiles are key-based safeguards, not compliance-grade DLP.
- v1.5.1 does not add a network sink, hosted dashboard, provider lookup, token counter, cost engine, or default upload path.
- Manual trace writing remains `schemaVersion: "0.1"`.

## Compatibility notes

- v0.1 trace files remain readable.
- v0.2 persisted-event files are accepted by inspection read paths through normalization but are not the default write format.
- Token usage additions are optional and additive. Existing input/output consumers remain compatible.
- v1.5.1 is intended as a patch release; no breaking API or package export removal is planned.
