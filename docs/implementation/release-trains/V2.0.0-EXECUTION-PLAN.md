# V2.0.0 Execution Plan

## Train

```yaml
train: "v2.0.0"
theme: "Stable local trace utility contract"
executionMode: "autonomous-release-train"
branch: "main"
releasePolicy: "RC-gated major contract train; do not publish stable without explicit maintainer release authorization"
```

## Goal

Turn the v1.x local-first tracing workbench into a clean v2 contract release with a small root API, stable schema 1.0 path, deterministic migration, and compatibility for existing v0.1/v0.2 traces.

## Product Boundaries

- Keep AgentInspect local-first, CLI-first, TypeScript-first, dependency-light, safe by default, and explicit about network behavior.
- Do not add hosted SaaS/APM, default telemetry upload, prompt/eval hosting, provider pricing, default replay/cassette execution, raw chain-of-thought capture, or default provider/network behavior.
- Do not publish v2 stable until release readiness passes and the maintainer explicitly authorizes the release.
- Existing v0.1 and v0.2 traces must remain readable.
- Breaking root export removals are allowed only inside the v2 contract chunks and must be documented with migration guidance.
- New schema 1.0 behavior must be bounded, redacted, deterministic, warning-rich, and non-mutating for existing traces.
- Private packages remain private unless explicitly authorized for first public publication.

## Deferred V1.9 Recovery

The v1.9.0 workflow partially published. `agent-inspect`, `@agent-inspect/ai-sdk`, `@agent-inspect/langchain`, and `@agent-inspect/tui` reached `1.9.0`; `@agent-inspect/openai-agents` remains published at `1.8.0` because npm returned `E404` during `@agent-inspect/openai-agents@1.9.0` publication. The maintainer acknowledged that npm auto-publish/trusted publishing for OpenAI Agents will be fixed manually later.

Do not attempt to recover or republish `@agent-inspect/openai-agents@1.9.0` inside this v2 train unless the maintainer explicitly reopens that recovery item.
