# v4.0.0 — Local Trace Workspace — release readiness

## Summary

First release of the **local trace workspace**: a project-local layout + manifest (`.agent-inspect/workspace.json`) with `workspace` CLI commands and a new experimental `agent-inspect/workspace` subpath export. Additive and backward-compatible — existing trace directories keep working and trace files are never deleted.

- **Baseline:** 3.5.5
- **This release:** 4.0.0 (linked bump across all 16 published packages)
- **Type:** new product surface (core + CLI), new gated subpath export

## Public surface changes

- New subpath export `agent-inspect/workspace` (and internal `@agent-inspect/core/workspace`).
- Wired: root `package.json` exports, `packages/core/package.json` exports, `entries/workspace.ts`, `tsup.core.config.ts`, `tsconfig.json` paths, `vitest.config.ts` alias/noExternal.
- Stability tests updated: `subpath-exports.test.ts`, `package-exports-compat.test.ts`, `api-stability.test.ts`.
- Root API value exports unchanged (allowlist stable).

## Safety

- Manifest parsing validates shape and bounds size; invalid input rejected, never thrown into user code.
- All manifest-derived paths resolved and confirmed inside the workspace directory (traversal guard, with unit tests).
- `workspace clean` is dry-run by default; trace directories are never targeted.
- No network I/O, no upload, no daemon, no database dependency.

## Out of scope (confirmed)

- No SQLite/DB in root or core (index remains disabled by default; SQLite index arrives in v4.1 as a separate optional package).
- No migration of existing traces by default.
- No cloud sync / hosted account / upload.
