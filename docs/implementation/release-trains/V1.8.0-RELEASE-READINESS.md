## Scope

- experimental `agent-inspect/checks` rule engine with run/tool/LLM/structure/safety/baseline rules;

## Compatibility

- Stable root imports remain unchanged and root API surface remains small.
- New check APIs are exposed through the experimental `agent-inspect/checks` subpath.
- Writer APIs remain under `agent-inspect/writers`; reader APIs remain under `agent-inspect/readers`; low-level runtime remains under `agent-inspect/advanced`.
- Manual traces still write `schemaVersion: "0.1"` by default.
- v0.1 and v0.2 AgentInspect JSONL traces remain readable.
- OpenInference and OTLP JSON remain compatibility reader/export formats, not a third AgentInspect persisted schema.
- Node support remains `>=20`.
- Root/core has no runtime dependency on AI SDK, OpenAI Agents, LangChain, Vitest, Jest, OpenTelemetry, or hosted services.

## Security And Network

- No default telemetry upload, hosted ingestion, GitHub API comments, provider calls, replay/cassette runner, or repository-write automation was added.
- Checks, safety verification, baseline comparison, and artifacts operate over existing local reader output.
- Safe artifact outputs include structural counts, statuses, bounded findings, diagnostics, and evidence paths; they avoid raw prompts, outputs, request/response bodies, headers, secrets, API keys, and full tool payloads.
- Reporter artifacts are structural and require explicit trace association; they do not infer trace relationships from timestamps and do not read trace contents into artifacts.
- `scan` and `verify-safe` are best-effort local safety checks, not compliance certification.
- Redaction remains key-based and bounded; generated files still require human review before broad sharing.
