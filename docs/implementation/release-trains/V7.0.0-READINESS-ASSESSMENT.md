# V7.0.0 — Readiness Assessment (Conditional)

**Train:** v7 (conditional ecosystem + intelligence layer)
**Status:** NOT scheduled. Gates recorded here, not executed.
**Authority:** [../ROADMAP_V3_5_TO_V7.md](../ROADMAP_V3_5_TO_V7.md) (§ v7)

> v7 is conditional. Do not execute any v7 chunk until the adoption gates below are met and a maintainer explicitly authorizes a v7 release train. This document records the entry criteria and candidate scope only.

## Entry gates (all must be true before scheduling v7)

- Teams use workspaces and suites repeatedly.
- Self-hosted Studio has real users.
- External plugins/adapters appear.
- MCP/coding-agent workflows are used.
- Users request extension/intelligence features rather than setup help.

## Candidate scope (recorded, not committed)

- **Trace intelligence plugins:** LLM-assisted explain, local-only explain, workflow analyzers, browser-agent analyzer, MCP analyzer, support-agent analyzer.
- **Context optimization research:** `@agent-inspect/context`, trace-aware context usage, compression decisions as trace events, token-budget checks (no cloud compression service).
- **Browser-agent support:** `@agent-inspect/browser`, `@agent-inspect/stagehand`, `@agent-inspect/playwright`.
- **Enterprise packaging:** Docker Compose self-hosted bundle, optional Helm chart, Postgres mode, internal auth mode, offline docs, air-gapped instructions.
- **External extension registry:** adapter package metadata, conformance status, privacy review, maintainer, supported framework versions.

## Non-goals (hard)

- No paid cloud
- No multi-tenant SaaS
- No default telemetry
- No cost platform
- No prompt registry
- No dataset management platform
- No automatic remediation

## Hard stop-gates for any future v7 work

- Every new public package first publication + Trusted Publishing is a manual maintainer gate.
- Any LLM-assisted feature must default to local-only / opt-in, no hidden network, redaction enforced.
- Any network/auth/enterprise packaging behavior requires security review.
- No schema break; additive only.

## Assessment outcome (this pass)

**Not ready to schedule v7.** v6.1–v6.4 release trains are complete in-repo; adoption gates (repeated workspace/suite use, self-hosted Studio users, external plugins, MCP workflow usage) are **not yet measured**. Revisit when npm adoption signals and maintainer review confirm entry criteria.

**Recorded baseline after v6.4:** plugin convention, MCP coding-agent tools, standards graduation docs/recipes shipped @ 6.4.0.

## No-upload / no-SaaS confirmation

Confirmed: v7 remains conditional and, if pursued, stays local/self-hosted with no paid cloud, no SaaS, and no default telemetry.
