# V4.2.0 — Sessions and Activity

**Train:** v4.2.0
**Theme:** Sessions, conversations, retries, attempts, handoffs, and activity as first-class local concepts.
**Authority:** [../ROADMAP_V3_5_TO_V7.md](../ROADMAP_V3_5_TO_V7.md) (§ v4.2.0)

## Release goal

Let developers understand multi-run agent activity without opening every trace, using explicit metadata (sessionId/correlationId/groupId/jobId/workflowId/handoff/attempt/retryOf) and confidence labels — never timestamp-only causality.

## Product boundary

Local-first CLI. No invented parent-child relationships. Unknown relationships stay visible.

## Non-goals

- No timestamp-only causality inference
- No network / upload
- No hosted session store

## Ordered chunks

- `v4.2-0` Session/activity RFC refresh.
- `v4.2-1` Session model + core aggregation.
