## Scope

v2.0.0 is the contract-stabilization train. It prepares the first v2 release around:

- schema 1.0 persisted InspectEvent rows for `createInspector()` and built-in writer/runtime output;
- backwards-compatible v0.1/v0.2/v1.0 local trace reads;
- explicit `agent-inspect migrate <input> --to 1.0 --dry-run` and `--output <file>` migration workflow;
- small stable root value API with advanced/reader/writer/exporter/check helpers kept on subpaths;
- public docs aligned to the v2 root/subpath and migration contract.

## Release Notes Draft

Draft notes for the maintainer release-prep step:

### Added

- Added schema 1.0 persisted InspectEvent fixtures, validation, and writer/runtime output paths.
- Added `agent-inspect migrate` for explicit single-file dry-run and output migration to schema 1.0.
- Added root API allowlist enforcement for the v2 contract.

### Changed

- `createInspector()` and built-in persisted writer paths now target schema 1.0 rows.
- Public docs now describe the v2 small root API, subpath ownership, and explicit migration workflow.
- Reader compatibility covers local v0.1, v0.2, and v1.0 AgentInspect JSONL inputs.

### Safety

- Manual/global tracing remains `schemaVersion: "0.1"` for 1.x compatibility.
- Migration is explicit and non-destructive; no automatic trace rewrite or in-place migration was added.
- No hosted upload, telemetry, provider call, replay/cassette execution, or network behavior was added.

## Compatibility

- Global/manual writes remain `schemaVersion: "0.1"`.
- v0.1, v0.2, and v1.0 AgentInspect JSONL traces remain readable.
- The stable v2 root value API is limited to `inspectRun`, `step`, `maybeInspectRun`, `observe`, `createInspector`, `createFileWriter`, `readTrace`, `readTraceFile`, `readTraceEvents`, `renderReport`, and `redactValue`.
- Advanced/runtime, persisted conversion, logs, exporters, diff, writers, readers, and checks helpers remain on their subpaths.
- ESM, CJS, declarations, CLI behavior, and Node `>=20` remain valid.
- Root/core still has no runtime dependency on AI SDK, OpenAI Agents, LangChain, Vitest, Jest, OpenTelemetry, provider SDKs, or hosted services.

## Security And Network

- No default telemetry upload, hosted ingestion, provider call, replay/cassette runner, GitHub API write, npm publication, tag, or GitHub release was added.
- Schema 1.0 write paths preserve local-first behavior and keep writer failures isolated through existing diagnostics/stats contracts.
- Migration remains local, explicit, and read/write-file based; it does not upload traces or perform remote conversion.
- No raw chain-of-thought capture, provider pricing, cost engine, destructive migration, or weaker redaction/escaping/traversal/size policy was added.
