# v4.2.0 — Sessions and Activity — release readiness

## Summary

Makes sessions, activity, handoffs, retries, and errors first-class local debugging concepts on top of the v2.4 session vocabulary. Adds derived session status, activity summaries, expanded CLI subcommands, and optional SQLite index acceleration with scan fallback. Additive — no trace schema change, no invented relationships.

- **Baseline:** 4.1.0
- **This release:** 4.2.0 (linked bump across all 17 published packages)
- **Type:** core session enrichment + CLI expansion + optional index acceleration

## Public surface changes

- `agent-inspect/advanced`: exports `deriveSessionStatus`, `enrichSessionSummary`, `buildActivitySummary`, `renderActivitySummaryHuman`, and v4.2 session/activity types.
- Extended `SessionSummary` with `status`, timing, `lastError`, `retryCount`, `lastActivity`, etc.
- CLI: `sessions` command group gains subcommands; bare `agent-inspect sessions` still lists (backward compatible); `agent-inspect session <id>` unchanged.
- Optional index acceleration via `@agent-inspect/index-sqlite` when installed and healthy; always enriches from trace files for handoff/retry parity.

## Safety

- No timestamp-only causality; v2.4 confidence/warning rules preserved.
- Index path degrades to scan on missing package, stale/corrupt index, or errors.
- Read-only; trace files never mutated. No network.
