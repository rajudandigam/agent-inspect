# V4.0.0 — Local Trace Workspace

**Train:** v4.0.0
**Theme:** Move from command collection to project-local trace workspace.
**Authority:** [../ROADMAP_V3_5_TO_V7.md](../ROADMAP_V3_5_TO_V7.md) (§ v4.0.0)

## Release goal

Introduce a stable local workspace layout and manifest (`.agent-inspect/workspace.json`) plus `workspace` CLI commands, so a project's local agent traces form a coherent workbench (runs, reports, artifacts, bundles, index, notes).

## Product boundary

Local-first, CLI-first. Workspace commands are read-only unless the user explicitly confirms a destructive action. JSONL remains the source of truth.

## Non-goals

- No daemon
- No database dependency (root stays light)
- No cloud sync / hosted account
- No migration of existing traces by default
- No upload

Full readiness matrix (build, typecheck, test, coverage, fixtures, recipes, size, test:all, pack:smoke, compat:smoke, `npm pack --dry-run`). Verify old traces readable, existing dirs adoptable, no trace mutation without explicit confirmation, no root DB dependency, package contents intended, docs aligned.

- New core subpath export (`agent-inspect/workspace`) requires root export review (package.json exports, `entries/`, tsup config, stability tests).

## Migration / backward-compatibility notes

`workspace.json` is additive (`schemaVersion "1.0"`). Existing `.agent-inspect` dirs are adopted, not rewritten. No destructive migration.

## Package impacts

`agent-inspect` (core + CLI). Possible new core subpath export (gated).

## Root dependency impacts

None expected. No SQLite/DB in root.

## Security / network impacts

Path-traversal guards for workspace paths. No network. No upload.

## No-upload / no-SaaS confirmation

Confirmed: local-only, no upload, no hosted service.
