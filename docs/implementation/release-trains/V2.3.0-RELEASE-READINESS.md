## Scope

v2.3.0 is the framework adapter hardening train. It improves the official adapter paths before adding new packages:

- `@agent-inspect/ai-sdk` hardened around explicit AI SDK telemetry, route-style factories, per-request integration isolation, streaming/tool/error/parallel fixtures, token metadata, and metadata-only capture defaults;
- `@agent-inspect/openai-agents` hardened around local-only replacement processors, advanced additional processor wording, no default upload confusion, and broader no-network span fixtures;
- `@agent-inspect/langchain` hardened for LangGraph-shaped metadata through the existing LangChain callback boundary, including graph/node identity, subgraphs, checkpoint/session IDs, stream modes, handoffs, and parallel branch hints;
- shared adapter conformance evidence now has a fixture manifest and release gate;
- Mastra and NestJS framework packages remain explicitly deferred. NestJS remains covered through structured-log ingestion, not a framework adapter package.

No runtime source changes are made in this readiness chunk. Adapter behavior was implemented in earlier v2.3 chunks and is documented/readiness-recorded here.

## Release Notes Draft

Draft notes for the release-prep step:

### Added

- Added a route-style AI SDK telemetry recipe for per-request integration factories while preserving metadata-only capture defaults.
- Added a shared adapter conformance fixture manifest and release evidence gate for official adapters.

### Changed

- Hardened `@agent-inspect/ai-sdk` lifecycle, streaming, tool, parallel, abort/error, token metadata, and safe telemetry documentation.
- Hardened `@agent-inspect/openai-agents` local-only replacement processor guidance and fixture coverage for agents, generations, tools, handoffs, guardrails, response, MCP tools, custom, transcription, and speech span shapes.
- Improved LangGraph-through-LangChain metadata mapping for graph nodes, subgraphs, checkpoints, session/thread IDs, stream modes, handoffs, and parallel branches without adding a new LangGraph package.
- Updated README, adapter docs, conformance docs, and recipe notes to make the supported adapter paths and deferred Mastra/Nest decisions explicit.

### Safety

- Official adapters remain explicit, opt-in, local-first, and metadata-only by default.
- No root/core dependency on AI SDK, OpenAI Agents, LangChain, LangGraph, Mastra, NestJS, provider SDKs, OpenTelemetry, hosted services, or LLM providers was added.
- No default upload behavior, hidden telemetry, provider calls, raw chain-of-thought capture, replay/cassette runner, or hosted dashboard was added.
- Raw prompts, outputs, tool payloads, headers, request bodies, and response bodies remain excluded by default.

### Known Limits

- AI SDK preview capture options still diagnose and fall back to metadata-only capture until bounded preview persistence is implemented.
- OpenAI Agents `addTraceProcessor()` remains an advanced user-owned mode that may preserve existing/default processors.
- LangGraph support stays inside `@agent-inspect/langchain`; no separate `@agent-inspect/langgraph` package is shipped.
- Mastra and NestJS framework packages remain deferred until demand and extension-point evidence justify narrow local-only integration.

## Compatibility

- Existing root APIs and published imports remain valid.
- Global/manual writes remain compatible with existing trace readers.
- v0.1, v0.2, and v1.0 AgentInspect traces remain readable.
- Schema 1.0 persisted runtime/writer compatibility remains unchanged.
- Adapter package APIs remain experimental and additive.
- ESM, CJS, declarations, CLI behavior, and Node `>=20` remain valid.
- Root/core still has no runtime dependency on framework SDKs, test runners, provider SDKs, hosted services, or LLM providers.

## Security And Network

- No default telemetry upload, hosted ingestion, provider call, replay/cassette runner, GitHub API write, npm publication, tag, or GitHub release was added.
- Official adapters write only to explicit local writers or `traceDir` paths.
- Recipes and conformance fixtures are no-network and dependency-isolated.
- No schema, persistence, traversal, escaping, size-limit, or redaction relaxation was introduced.
