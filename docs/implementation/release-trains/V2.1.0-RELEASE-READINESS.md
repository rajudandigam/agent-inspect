## Scope

v2.1.0 is the utility-triangle train. It adds deterministic local eval and redaction workflows around the existing trace workbench:

- optional public package `@agent-inspect/redact`;
- optional public package `@agent-inspect/eval`;
- root CLI commands `agent-inspect redact` and `agent-inspect eval`;
- shared redaction profiles/findings used by trace safety, export, explain, artifacts, docs, and recipes;
- deterministic local eval checks for success, required/forbidden tool usage, and grounding heuristics;
- adoption recipes for local eval, share-safe redaction, and CI artifacts.

No runtime source changes were made in this readiness chunk. The only package configuration change in this chunk aligns Changesets release metadata with the v2.1 public/private package set.

## Release Notes Draft

Draft notes for the maintainer release-prep step:

### Added

- Added `@agent-inspect/redact`, a deterministic local redaction utility with findings, profiles, and custom detector support.
- Added `@agent-inspect/eval`, a deterministic local eval utility for trace-backed success, tool-use, and grounding checks.
- Added `agent-inspect redact` and `agent-inspect eval` CLI workflows.
- Added recipes for local eval checks, share-safe redaction, and CI artifact workflows.

### Changed

- AgentInspect docs now present the v2.1 utility triangle: trace, eval, and redact.
- Safe sharing, CLI, API, adapters, comparison, limitations, and known-issues docs now reference eval/redact where relevant.
- Package smoke now covers `@agent-inspect/redact` and `@agent-inspect/eval` as public optional packages.

### Safety

- Eval is deterministic and local by default; no LLM judge, hosted eval platform, provider call, or network behavior was added.
- Redaction is deterministic and local; it does not make compliance guarantees.
- No raw chain-of-thought capture, replay/cassette execution, default telemetry upload, or provider pricing behavior was added.

## Compatibility

- Existing root APIs and published imports remain valid.
- Global/manual writes remain `schemaVersion: "0.1"`.
- v0.1, v0.2, and v1.0 AgentInspect traces remain readable.
- Schema 1.0 persisted runtime/writer compatibility remains unchanged.
- `@agent-inspect/redact` and `@agent-inspect/eval` are optional package surfaces; no new root exports were added.
- ESM, CJS, declarations, CLI behavior, and Node `>=20` remain valid.
- Root/core still has no runtime dependency on AI SDK, OpenAI Agents, LangChain, Vitest, Jest, OpenTelemetry, provider SDKs, hosted services, or LLM providers.

## Security And Network

- No default telemetry upload, hosted ingestion, provider call, replay/cassette runner, GitHub API write, npm publication, tag, or GitHub release was added.
- Redaction and eval workflows are local-only and deterministic by default.
- Redaction findings preserve explicit detector/action/severity metadata without claiming regulatory compliance.
- Eval grounding checks use caller-supplied answer/context/source data; they do not fetch remote context or call a model.
- No schema, persistence, traversal, escaping, or size-limit relaxation was introduced.
