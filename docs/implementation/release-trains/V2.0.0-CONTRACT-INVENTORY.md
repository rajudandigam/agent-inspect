## Current Root Surface

`packages/core/src/index.ts` currently re-exports stable runtime helpers plus persisted converters, log ingestion, storage, readers, writers, diff, terminal rendering, safety helpers, report/explain/stats/search, exporters, and advanced runtime/context utilities.

The v2 stable root target is:

```ts
import {
  createInspector,
  inspectRun,
  maybeInspectRun,
  step,
  observe,
  getCurrentCorrelationMetadata
} from "agent-inspect";
```

Root type exports should remain limited to essential stable user-facing contracts for those functions and their event model:

- `StepType`, `StepStatus`, `RunStatus`;
- `ErrorInfo`, `TokenMetadata`, `StepMetadata`;
- `Run`, `Step`, `TraceCorrelationMetadata`;
- `InspectRunOptions`, `StepOptions`, `ObserveOptions`;
- `CreateInspectorOptions`, `Inspector`, `InspectorCaptureOptions`, `InspectorObserveOptions`, `InspectorRunOptions`, `InspectorStepOptions`;
- `RedactionProfile`.

Everything else with a public home must move to a subpath for v2 root imports. This is the intended v2 breaking change and should be enforced by shrinking `ROOT_VALUE_EXPORTS_ALLOWLIST` in `packages/core/test/api-stability.test.ts`.

## Current Persisted Contracts

### v0.1 manual JSONL

Source: `packages/core/src/types.ts`.

- `schemaVersion: "0.1"`.
- Rows are lifecycle events: `run_started`, `run_completed`, `step_started`, `step_completed`.
- Core/global helpers and LangChain persistence still write v0.1 during v1.x.
- v2 readers must continue to read v0.1 and migration must never overwrite originals by default.

### v0.2 persisted inspect event

Source: `packages/core/src/types/persisted-inspect-event.ts`.

- `schemaVersion: "0.2"`.
- Required fields: `eventId`, `runId`, `kind`, `name`, `timestamp`, `confidence`, `source`.
- Optional fields: `parentId`, `status`, `startedAt`, `endedAt`, `durationMs`, `attributes`, `inputSummary`, `outputSummary`, `error`, `tokenUsage`, `trace`.
- Used by AI SDK, OpenAI Agents, `createInspector`, readers, writers, checks, exporters, and normalized tree building.
- v2 schema 1.0 must evolve this shape rather than introduce a third unrelated model.

### v1.0 frozen target

Source: `docs/proposals/STABLE-SCHEMA-1.0.md`.

- `schemaVersion: "1.0"`.
- Default v2 writer output is JSONL.
- One row represents one inspectable event/node where possible.
- Required and optional fields follow the v0.2 baseline.
- Unknown optional fields are preserved by readers/migration where safe and otherwise surfaced through warnings or `unsupportedFields`.
- Source details use coarse `source.type` and framework/package detail in `source.name`/`source.version`.
- No default raw prompt/output/body/chain-of-thought capture; summaries remain bounded and explicit.
- No core provider pricing or billing field.

## Readers, Writers, Checks, Exporters

- Readers currently resolve input once, detect AgentInspect v0.1/v0.2 JSONL plus OpenInference JSON and OTLP JSON, normalize to `PersistedInspectEvent[]`, preserve warnings, and bound input to 10 MiB.
- Writers currently accept `PersistedInspectEvent`, prepare/redact/bound events before storage, isolate write/flush/close failures, expose stats, and perform no network I/O.
- Checks currently consume `TraceReadResult` and normalized facts; rules are synchronous, pure, local, and evidence-based.
- Exporters currently consume normalized run trees and produce deterministic local markdown/html/OpenInference/OTLP outputs with redaction helpers.

V2 implementation should update these contracts in place for schema 1.0 rather than creating parallel systems.

## Intended V2 Breaking Changes

These are the planned breaking changes identified by this inventory:

1. The root value export set shrinks to `createInspector`, `inspectRun`, `maybeInspectRun`, `step`, `observe`, and `getCurrentCorrelationMetadata`.
2. Advanced/root-adjacent values remain available only through documented subpaths.
3. Default v2 built-in writer output changes from v0.1/v0.2-era writes to schema 1.0 JSONL.
4. Public docs and consumer fixtures must stop teaching advanced root imports.

No other breaking change is authorized by this inventory.
