# v2.3.0 Execution Plan

## Train

```yaml
train: "v2.3.0"
theme: "Framework adapter hardening"
baselineVersion: "previous published train"
executionMode: "codex-autonomous-release-train"
```

## Goal

Make official adapters genuinely low-friction and fixture-backed, without adding weak new adapters.

## Product boundaries

This train must preserve:

- local-first behavior;
- no default network calls;
- no root/core framework or provider dependencies;
- no raw chain-of-thought capture;
- no provider pricing engine;
- no hosted service or dashboard;
- deterministic behavior by default;
- old trace readability;
- package ESM/CJS/declaration compatibility.

## Train validation

Run the appropriate focused gates per chunk. Before release readiness, run:

```bash

pnpm build
pnpm typecheck
pnpm test
pnpm test:coverage
pnpm fixtures:check
pnpm recipes:check
pnpm size
pnpm test:all
pnpm pack:smoke
pnpm compat:smoke
npm pack --dry-run
git diff --check
```

Additional train-specific validation:

- all official adapters pass executable conformance;
- no root/core dependency increase;
- package smoke for all public optional adapters;
- no-network default tests.
