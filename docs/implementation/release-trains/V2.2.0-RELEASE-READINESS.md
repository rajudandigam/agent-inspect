## Scope

v2.2.0 is the test reporter and CI workflow train. It prepares AgentInspect for daily test loops without adding hosted behavior:

- shared experimental `agent-inspect/reporters` manifest helpers;
- workspace Vitest reporter behavior for local failure artifacts;
- workspace Jest reporter behavior for local failure artifacts;
- root CLI `agent-inspect ci-summary` for deterministic CI summaries from reporter manifests;
- GitHub Actions artifact recipe updates that rely on user-controlled artifact upload;
- public docs for reporter artifacts, local summaries, safety boundaries, and release gates.

No runtime source changes are made in this readiness chunk. Reporter behavior was implemented in earlier v2.2 chunks and is only documented/readiness-recorded here.

## Release Notes Draft

Draft notes for the maintainer release-prep step:

### Added

- Added shared experimental `agent-inspect/reporters` helpers for reporter manifest creation, safe relative artifact paths, diagnostics, and deterministic ordering.
- Added adoption-ready workspace Vitest reporter artifacts for failed tests with explicit trace associations and quiet success defaults.
- Added adoption-ready workspace Jest reporter artifacts for failed tests with explicit trace associations and CJS/Jest compatibility coverage.
- Added `agent-inspect ci-summary` to summarize local reporter manifests into deterministic Markdown or JSON for CI logs and step-summary files.
- Updated GitHub Actions artifact recipes to demonstrate user-controlled upload of local AgentInspect artifacts and reporter summaries.

### Safety

- Reporter artifacts are local-only and metadata-bounded by default.
- `ci-summary` reads reporter manifest JSON only. It does not read trace contents, rerun tests, upload artifacts, call GitHub APIs, require repository write access, or mutate repository state.
- Reporter artifact paths are validated as safe relative paths before summary rendering.
- No raw chain-of-thought capture, hosted dashboard, telemetry upload, provider call, GitHub API writer, or artifact upload behavior was added.

### Known Limits

- `@agent-inspect/vitest` and `@agent-inspect/jest` require manual `2.1.0` bootstrap publication before Trusted Publishing and normal v2.2 release prep.
- Reporter manifests are experimental `schemaVersion: "0.1"` local artifact indexes, not a hosted ingestion schema.

## Compatibility

- Existing root APIs and published imports remain valid.
- Global/manual writes remain `schemaVersion: "0.1"`.
- v0.1, v0.2, and v1.0 AgentInspect traces remain readable.
- Schema 1.0 persisted runtime/writer compatibility remains unchanged.
- `agent-inspect/reporters` is an additive experimental subpath.
- `agent-inspect ci-summary` is an additive CLI command.
- ESM, CJS, declarations, CLI behavior, and Node `>=20` remain valid.
- Root/core still has no runtime dependency on AI SDK, OpenAI Agents, LangChain, Vitest, Jest, OpenTelemetry, provider SDKs, hosted services, or LLM providers.

## Security And Network

- No default telemetry upload, hosted ingestion, provider call, replay/cassette runner, GitHub API write, npm publication, tag, or GitHub release was added.
- Reporter packages remain optional workspace packages and do not leak framework dependencies into root/core.
- Reporter summaries avoid raw prompt/output/request/response/header/API-key/secret/tool payload values.
- No schema, persistence, traversal, escaping, size-limit, or redaction relaxation was introduced.
