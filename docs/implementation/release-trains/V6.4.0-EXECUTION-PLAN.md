# V6.4.0 — Standards Graduation

**Train:** v6.4.0
**Theme:** Graduate exports to standards-based observability (no vendor SDK matrix in core).
**Authority:** [../ROADMAP_V3_5_TO_V7.md](../ROADMAP_V3_5_TO_V7.md) (§ v6.4.0)

## Release goal

Provide standards-based interop: OpenInference fixture validation, an OTLP JSON exporter, and import/interop recipes (Langfuse, Phoenix) plus vendor docs (New Relic, Datadog, Honeycomb), with OpenTelemetry semantic-convention pinning. Prefer standards over vendor SDKs; no default upload; no managed collector.

## Product boundary

Local/self-hosted export flows. Standards-first. No vendor SDK matrix in core. No default upload.

## Non-goals

- No vendor SDK matrix in core
- No default upload
- No managed collector
