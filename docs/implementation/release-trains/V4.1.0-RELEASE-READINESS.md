# v4.1.0 — Optional Local Index — release readiness

## Summary

First release of the **optional local SQLite index**: a disposable, rebuildable index over AgentInspect JSONL traces that accelerates local queries. Ships as a new optional package `@agent-inspect/index-sqlite` plus `agent-inspect index sqlite` CLI subcommands. JSONL remains the source of truth; the index is always safe to delete and trace files are never mutated. SQLite lives only in the optional package — never in root/core.

- **Baseline:** 4.0.0
- **This release:** 4.1.0 (linked bump across all 17 published packages)
- **Type:** new optional package + additive CLI surface

## Public surface changes

- New optional public package `@agent-inspect/index-sqlite` with exports: `buildIndex`, `rebuildIndex`, `cleanIndex`, `resolveIndexDbPath`, `indexStatus`, `isIndexStale`, `queryRuns`, and supporting types.
- New CLI subgroup `agent-inspect index sqlite` (loaded on demand; prints an install hint when the optional package is absent).
- Root `agent-inspect` package: CLI bundle gains the new subcommands (shipped via `packages/cli/dist`).
- No change to existing root/core value exports or the trace JSONL schema.
- `@agent-inspect/index-sqlite` declares a regular `dependencies` entry on `agent-inspect` (matching sibling packages), not a peer dependency.

## Safety

- Index is derived from JSONL and always safe to delete; trace files are never mutated.
- Read APIs are non-throwing: missing/corrupt index yields empty results (`queryRuns`), `healthy: false` (`indexStatus`), or `true` (`isIndexStale`) so callers fall back to a directory scan.
- Corruption is detected via `integrity_check`; a failed check is treated as absent and a rebuild recreates the database.
- Index paths resolved under the trace directory; `clean` removes only the DB and its WAL/SHM sidecars.
- No network I/O, no upload, no daemon. SQLite (`better-sqlite3`) is confined to the optional package; root/core add no database dependency.
