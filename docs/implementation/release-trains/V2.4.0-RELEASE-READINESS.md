## Scope

v2.4.0 is the sessions and MCP client telemetry train. It adds multi-run workflow navigation without schema breaks or gateway products:

- additive session/workflow metadata model and `agent-inspect/advanced` session index helpers;
- `agent-inspect sessions` and `agent-inspect session` CLI (timeline, critical-path, diagnostics, JSON);
- session-aware `search --session` and `check --session` / `--group` with aggregated evidence;
- new public optional `@agent-inspect/mcp` for local MCP **client** `tools/list` and `tools/call` tracing;
- fixtures under `fixtures/sessions/` and recipe [mcp-client-tracing](../../../examples/recipes/mcp-client-tracing/).

Out of scope: MCP server/gateway, timestamp-only causality, schema version changes, viewer, upload pipelines.

## Release Notes Draft

### Added

- Session/workflow causality RFC and additive metadata guidance in `docs/SCHEMA.md`.
- `buildSessionIndex`, scope/cohort helpers, and session fixtures on `agent-inspect/advanced`.
- CLI: `sessions`, `session`, `search --session`, `check --session` / `--group`.
- Public `@agent-inspect/mcp` with `wrapMcpClient()` for local MCP client telemetry.
- Recipe: `examples/recipes/mcp-client-tracing`.

### Changed

- README, CLI, ADAPTERS, recipes index, and ROADMAP document session and MCP client boundaries.

### Safety

- Local-first, read-only session/MCP tooling; no default network calls.
- No MCP gateway/server, no schema break, no timestamp-only causality inference.
- MCP summaries are bounded; raw tool payloads are not captured by default.

### Known Limits

- Cohort CLI (`agent-inspect cohort`) remains deferred; grouping helpers ship on `advanced` only.
- `@agent-inspect/mcp` uses duck-typed `McpClientLike`; official SDK types are not required at runtime.
- Session handoff/retry edges require explicit metadata; correlated inference emits warnings.

## Compatibility

- v0.1, v0.2, and v1.0 traces remain readable without migration.
- Root stable APIs unchanged; session helpers are on `agent-inspect/advanced` only.
- Existing search/check behavior unchanged without session/group flags.
- ESM, CJS, declarations, CLI behavior, and Node `>=20` remain valid.
