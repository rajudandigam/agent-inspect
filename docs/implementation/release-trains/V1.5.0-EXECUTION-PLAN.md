## Chunk 7 — Canonical dual-format read path

| Field | Value |
|-------|-------|
| **Milestone** | M4 |
| **Type** | refactor |
| **Depends on** | Chunk 6 |
| **Validation** | runtime |

### Goal

Single core read pipeline that accepts v0.1 `TraceEvent` JSONL and v0.2 `PersistedInspectEvent` JSONL, normalizing to a common in-memory model for inspection.

### In scope

- `readTraceEvents` evolution or successor API
- Wire existing v0.2 converters/validators

### Out of scope

- Changing default write format (stays v0.1 for manual traces)

- `fixtures/traces-v0.2/`, tests

### Tests required

- Dual-format read tests
- `pnpm fixtures:check`
- v0.1 backward-compat tests unchanged/green

### Suggested commit message

```
refactor(core): canonical dual-format trace read path
```
