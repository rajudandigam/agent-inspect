# v1.7.0 Execution Plan

**Train:** v1.7.0 framework-native adoption
**Baseline:** published v1.6.0 runtime/reader/writer foundation
**Status:** planning
**Release policy:** chunked commits to `main`; no version bump, changeset, tag, publish, dependency addition, or adapter runtime code until the relevant chunk explicitly authorizes it.

## External API Baseline

Current official docs checked during planning reset:

- AI SDK v6 telemetry docs: `https://ai-sdk.dev/docs/ai-sdk-core/telemetry`
  - telemetry is experimental;
  - `experimental_telemetry.integrations` accepts integrations for `generateText` and `streamText`;
  - integration errors are caught by the AI SDK and do not break generation flow;
  - inputs and outputs are recorded by default unless `recordInputs: false` and `recordOutputs: false` are set.
- OpenAI Agents JS tracing docs: `https://openai.github.io/openai-agents-js/guides/tracing/`
  - default tracing can export to the OpenAI backend in supported server runtimes;
  - `addTraceProcessor()` adds processors in addition to default export behavior;
  - `setTraceProcessors()` replaces default processors, avoiding backend export unless an exporting processor is included;
  - sensitive generation/function inputs and outputs require explicit capture controls before any local processor ships.

## Train Guardrails

- Do not add AI SDK, OpenAI Agents, LangGraph, OpenTelemetry, or LangChain dependencies to root/core.
- Keep adapters as optional packages.
- Keep privacy defaults metadata-only.
- For AI SDK, examples and implementation must set or document `recordInputs: false` and `recordOutputs: false`.
- For OpenAI Agents, document local-only processor behavior before implementation and avoid default backend export surprises.
- Do not add network upload behavior or default telemetry upload.
- Do not change public trace schema, package versions, or release metadata until the release-readiness chunk.
