## Scope

v1.9.0 is the adoption leverage train. It converts the v1.8 local-first foundation into easier repeated use:

- private workspace `@agent-inspect/harness` package with `defineTarget`, `createFixtureRunner`, and `runFromArgv()` for fixture-driven local agent runs;
- harness recipes and CLI ergonomics for target listing, fixture files, JSON stdin, expected-output comparison, JSON stdout, and trace/check summary stderr;
- experimental `agent-inspect explain` dry-run and deterministic local explanation path with redaction before payload generation and no provider/network call by default;
- explicit provider design gate for future explain provider work, including environment requirements, payload boundaries, and no-chain-of-thought policy;
- promoted adapter adoption paths for AI SDK, OpenAI Agents, LangChain, and LangGraph with no-network examples where possible;
- v2 root API slimming plan, migration guidance, and root export allowlist enforcement.

## Release Notes Draft

Draft notes for the maintainer release-prep step:

### Added

- Added the private `@agent-inspect/harness` workspace package for fixture-driven local agent runners.
- Added `agent-inspect explain` dry-run and deterministic local explanation output for local trace investigation without provider calls by default.
- Added harness recipes for basic fixture use, AI SDK tool-shaped use, and LangGraph/LangChain-shaped local runs.

### Changed

- Promoted adapter adoption docs for AI SDK, OpenAI Agents, LangChain, and LangGraph.
- Clarified OpenAI Agents local-only replacement versus additional processor behavior.
- Documented the intended small v2 root API and the 1.x-to-v2 import migration path.

### Safety

- Explain payloads are redacted before display or future provider submission.
- Provider/network explain behavior remains design-gated and unimplemented in this train.
- Harness and adapter recipes remain local-first and avoid live vendor calls by default.

## Compatibility

- Existing 1.x root imports remain valid; no root export was removed.
- The intended v2 root API set is documented, while advanced/root compatibility aliases remain available in 1.x.
- `agent-inspect/advanced`, `/persisted`, `/logs`, `/exporters`, `/diff`, `/writers`, `/readers`, and `/checks` subpaths remain valid.
- Manual trace writing still defaults to `schemaVersion: "0.1"`.
- v0.1 and v0.2 AgentInspect JSONL traces remain readable.
- Node support remains `>=20`.
- Root/core has no runtime dependency on AI SDK, OpenAI Agents, LangChain, Vitest, Jest, OpenTelemetry, provider SDKs, or hosted services.

## Security And Network

- No default telemetry upload, hosted ingestion, provider call, replay/cassette runner, GitHub API write, or release publication was added.
- Explain provider calls are not implemented; dry-run/local explanation is the shipped behavior from this train.
- Harness recipes and adapter examples are local-first and no-network unless the user's app code explicitly performs network work.
- No new persisted schema, destructive migration, raw chain-of-thought capture, provider pricing, or cost engine was added.
- Generated trace/share artifacts still require human review before broad sharing.
