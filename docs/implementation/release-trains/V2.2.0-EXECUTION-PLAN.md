# v2.2.0 Execution Plan

## Train

```yaml
train: "v2.2.0"
theme: "Test reporters and CI workflows"
baselineVersion: "previous published train"
executionMode: "codex-autonomous-release-train"
```

## Goal

Make AgentInspect part of daily tests by promoting Vitest/Jest reporters and CI artifact workflows.

## Product boundaries

This train must preserve:

- local-first behavior;
- no default network calls;
- no root/core framework or provider dependencies;
- no raw chain-of-thought capture;
- no provider pricing engine;
- no hosted service or dashboard;
- deterministic behavior by default;
- old trace readability;
- package ESM/CJS/declaration compatibility.

## Train validation

Run the appropriate focused gates per chunk. Before release readiness, run:

```bash
pnpm install --frozen-lockfile
pnpm build
pnpm typecheck
pnpm test
pnpm test:coverage
pnpm fixtures:check
pnpm recipes:check
pnpm size
pnpm test:all
pnpm pack:smoke
pnpm compat:smoke
npm pack --dry-run
git diff --check
```

Additional train-specific validation:

- package-specific clean install tests for `@agent-inspect/vitest` and `@agent-inspect/jest`;
- fixture tests proving original test failures are preserved;
- CI summary fixture tests.
