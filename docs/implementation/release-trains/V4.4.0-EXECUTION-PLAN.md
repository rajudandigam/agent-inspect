# V4.4.0 — Observed Outcomes

**Train:** v4.4.0
**Theme:** Track whether the external world changed as expected, not only whether a tool returned success.
**Authority:** [../ROADMAP_V3_5_TO_V7.md](../ROADMAP_V3_5_TO_V7.md) (§ v4.4.0)

## Release goal

Add `observeOutcome(name, {...})` and surface observed outcomes in report/check/search, so tool success is distinguishable from real-world outcome success.

## Product boundary

Local-first. Redaction and size bounds apply to observed evidence. No raw sensitive payloads by default.

## Non-goals

- No raw sensitive payload capture by default
- No schema break
- No network / upload
