# v1.4.0 implementation plan

**Theme:** CI artifacts, timeline, stats, and deterministic search — deepen local inspection workflows without a dashboard or vendor upload.

**Status:** Implemented locally on `main` — **not published**.

**Prerequisite:** v1.3.0 shipped (correlation metadata, redaction profiles, LangChain streaming metadata).

**Companion:** [ROADMAP.md](../../ROADMAP.md) · [V1.3.0-RELEASE-READINESS.md](./V1.3.0-RELEASE-READINESS.md)

---

## Goals

1. **Local CI artifact workflows** — documented recipes and export examples for attaching trace artifacts in CI (e.g. GitHub Actions upload); no reporter package in v1.4.0 Chunk 1 unless scope explicitly expands later.
2. **`agent-inspect timeline <runId>`** — chronological CLI view of a run (steps, durations, streaming metadata where present).
3. **`agent-inspect stats --dir ./.agent-inspect --since 7d`** — lightweight local aggregates (run counts, error rates, duration summaries) — read-only, no backend.
4. **`agent-inspect search`** — deterministic local trace search (status, kind/type, name, tool, duration, since).
5. **Keep all output local** — no SaaS, no default network upload, no hosted analytics.

## Planned chunks

### Chunk 1 — CI artifact recipe and trace export workflow polish

| Deliverable | Notes |
| ----------- | ----- |
| `docs/CI-ARTIFACTS.md` | How to enable tracing in CI, export share-safe artifacts, upload as build artifacts |
| `examples/recipes/github-actions-artifact/` | Sample workflow + `AGENT_INSPECT=1` example |
| Export examples | markdown / html / json artifact patterns using existing `agent-inspect export` |
| Polish | Fix tiny export/docs bugs only if audit finds gaps |

**Out of scope for Chunk 1:** `@agent-inspect/*` reporter package, runtime behavior changes beyond docs/fixtures, vendor telemetry upload.

**Prompt stub:** [prompts/v1.4.0-chunk-1-ci-artifacts.md](./prompts/v1.4.0-chunk-1-ci-artifacts.md)

### Chunk 2 — `agent-inspect timeline <runId>`

| Deliverable | Notes |
| ----------- | ----- |
| CLI command | Chronological step list with timestamps, durations, status |
| Input | Existing JSONL traces under `--dir` (default `.agent-inspect`) |
| Output | Terminal table or plain text — no web UI |
| Tests | CLI tests with fixtures; correlation/streaming metadata display where available |

**Out of scope:** Interactive TUI timeline, cross-run dashboard, remote fetch.

### Chunk 3 — `agent-inspect stats --dir ./.agent-inspect --since 7d`

| Deliverable | Notes |
| ----------- | ----- |
| CLI command | Aggregate run counts, error rates, duration percentiles (lightweight) |
| Filters | `--since`, optional `--correlation-id` / `--group-id` when useful |
| Output | Local stdout only |
| Tests | Fixture-backed stats assertions |

**Out of scope:** Cost engine, LLM judge, cohort regression engine (defer to v1.5.x), database storage.

### Chunk 4 — `agent-inspect search`

| Deliverable | Notes |
| ----------- | ----- |
| CLI command | Deterministic search over local JSONL traces |
| Filters | `--status`, `--kind`/`--type`, `--name`, `--tool`, `--duration`, `--since`, `--limit` |
| Output | Human table or `--json` — no full metadata dump |
| Tests | Core + CLI fixture tests |

**Out of scope:** Semantic/fuzzy search, index files, network.

---

## Non-goals (v1.4.0 train)

- SaaS backend or hosted dashboard
- SQLite or other DB as core dependency
- Hosted analytics or fleet aggregation
- Cost / token pricing engine in core
- LLM-as-judge eval pipeline
- OTLP HTTP sink or default vendor upload
- NestJS / logging bridge **packages** (recipes may continue in examples)
- Vercel AI SDK adapter
- Default `schemaVersion: "0.2"` file writing
- Replay / cassette engine
- CI reporter npm package (recipes/docs only in v1.4.0; reporter packages deferred to v1.4.x/v1.5.x)
- Deterministic `search` is in scope for v1.4.0 Chunk 4
