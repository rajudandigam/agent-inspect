# v2.0 Post-Publish Review and v2.1 Entry Decision

## Date

2026-06-27

## Status

This document should be created after verifying the live repository and registry.

Expected current state:

- `agent-inspect` is at `2.0.0`.
- `@agent-inspect/ai-sdk` is at `2.0.0`.
- `@agent-inspect/openai-agents` is at `2.0.0`.
- `@agent-inspect/langchain` is at `2.0.0`.
- `@agent-inspect/tui` is at `2.0.0`.
- `main` is expected to match the `agent-inspect@2.0.0` tag.
- `@agent-inspect/harness`, `@agent-inspect/vitest`, and `@agent-inspect/jest` remain private unless explicitly approved for publication.

## Product state

AgentInspect has crossed the v2 contract line:

- small stable root API;
- schema 1.0 persisted writer path;
- v0.1 / v0.2 / v1.0 reader compatibility;
- explicit non-destructive migration workflow;
- stable subpaths for advanced workflows;
- local-first checks, reports, safe artifacts, and adapters.

## Immediate gaps to fix before v2.1 implementation

### 1. README version language

README must no longer say the current npm release line is `1.9.x` or that v2 is unreleased.

Required update:

- current npm release: `2.0.0`;
- mention schema 1.0 and migration as shipped in v2;
- move v1.8/v1.9 notes into release history.

### 2. Public ROADMAP.md

ROADMAP.md must be updated from the pre-v2 state.

Required update:

- current release on npm: `2.0.0`;
- `Released recently`: v2.0.0;
- `Now`: v2.1 utility triangle (`@agent-inspect/eval`, `@agent-inspect/redact`);
- future sequence: v2.2 reporters, v2.3 adapter hardening, v2.4 sessions/MCP, v2.5 guardrails/circuit, v2.6 viewer/IDE/MCP surfaces, v3.0 conditional extension ecosystem.

### 3. Operational state

Update:

- `docs/implementation/RELEASE-TRAIN-STATE.md`
- `docs/implementation/CURRENT-TASK.md`

```yaml
baselineVersion: "2.0.0"
```

### 4. v2.0 release-readiness doc

`docs/implementation/release-trains/V2.0.0-RELEASE-READINESS.md` currently appears to be pre-publication/release-prep oriented. Add a post-publish section with registry, tag, GitHub release, and clean install evidence.

### 5. Package docs files

If README or npm package content links to these docs, the root `package.json` `files` list should include them or the links should be changed:

- `docs/SAFE-TRACE-SHARING.md`
- `docs/INSTALL-SMOKE-TEST.md`

Preferred fix: include them in `files`.

### 6. Root API contract docs

Align all docs with the actual v2 root surface.

The root API should remain small. Advanced helpers should remain on subpaths. Do not re-add advanced symbols to the root just to satisfy old docs.

### 7. Active roadmap filename

Create a clearer active roadmap name:

```text
docs/implementation/ROADMAP-V2.1-TO-V3.md
```

Keep older roadmap files as historical references or redirect them from implementation README.

Recommended first v2.1 chunk:

