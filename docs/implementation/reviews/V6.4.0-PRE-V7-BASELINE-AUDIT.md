## Threat model (v6.4.1 scope)

| Boundary | Risk | Mitigation target |
|----------|------|-------------------|
| Packed consumer | Broken init output, wrong CLI docs | Packed E2E, README fixes |
| MCP stdio | Raw trace data leaving process | Central `prepareMcpToolResult` with redaction + byte caps |
| Bundle/artifact writers | Path traversal, false SAFE status | Sanitized paths, real `assessOpenedTrace` |
| HTML renderers | Trace-derived XSS | Escape all trace fields; no unsafe `innerHTML` |
| Plugin manifests | Coerced unsafe flags | Strict boolean/enum parsing |
| Studio DB init | Race, false Postgres claims | Await mkdir; clear capability messaging |

## Acceptance rules (v6.4.1)

- No MCP response bypasses redaction and bounds
- No share-safe result marked SAFE without real assessment
- UNKNOWN / UNSAFE fail closed by default
- Raw run IDs cannot escape bundle directories
- Viewer/Studio cannot execute trace-provided HTML
- Plugin booleans are strict
- Invalid thresholds rejected clearly
- Packed five-minute path passes from npm tarball

## Out of scope (v6.4.1)

- Trace schema changes
- Suite all-skipped semantics (v6.5.0)
- Cohort tolerances (v6.5.0)
- Studio product pages (v6.6.0)
- Semantic standards fixtures (v6.7.0)
- v7 implementation
