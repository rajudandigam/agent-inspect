# v1.3.0 release train

Maintainer execution guide for the **post–v1.2.0** train: correlation metadata, redaction profiles, and LangChain streaming depth.

**Companion:** [CURSOR-MAINTAINER-ROADMAP.md](./CURSOR-MAINTAINER-ROADMAP.md) · [ROADMAP.md](../../ROADMAP.md)

**Prerequisite:** v1.2.0 persisted-event foundation shipped (2026-06-11). Manual trace writing remains `schemaVersion: "0.1"`; v0.2 is not written by default.

---

## Train scope (v1.3.0)

### In scope

| Chunk | Intent |
| ----- | ------ |
| **Correlation metadata** | Propagate `correlationId`, `requestId`, `decisionId` in manual metadata and persisted `attributes` / `trace` blocks where appropriate. |
| **Redaction profiles** | Named presets beyond default exact-key redaction; opt-in via `InspectRunOptions` or export config. |
| **Share-safe export defaults** | Export helpers document and optionally apply stricter redaction profiles. |
| **LangChain streaming design** | RFC + metadata-only streaming callback support ([#14](https://github.com/rajudandigam/agent-inspect/issues/14)); size bounds; no prompt capture by default. |
| **Docs / tests** | API.md, LIMITATIONS.md, adapter docs; tests for new metadata and redaction behavior. |

### Out of scope (defer to later trains)

- Default `schemaVersion: "0.2"` file writing
- Storage dual-read / CLI read-path migration for v0.2
- `timeline` / `stats` / cohort CLI ([#11](https://github.com/rajudandigam/agent-inspect/issues/11), [#12](https://github.com/rajudandigam/agent-inspect/issues/12))
- CI reporters ([#24](https://github.com/rajudandigam/agent-inspect/issues/24))
- NestJS / logging bridge packages
- Vercel AI SDK adapter ([#30](https://github.com/rajudandigam/agent-inspect/issues/30))
- OTLP HTTP sink / vendor upload
- Replay engine, cost engine, SaaS/dashboard

## Suggested PR sequence

| PR | Title (working) | Notes |
| -- | --------------- | ----- |
| **3A** | Correlation metadata types and manual trace propagation | Additive metadata keys only; `0.1` schema compatible |
| **3B** | Redaction profile presets | Named profiles; default profile unchanged |
| **3C** | Share-safe export profile wiring | Export path only; no upload |
| **3D** | LangChain streaming RFC + metadata-only hooks | Experimental adapter surface; [#14](https://github.com/rajudandigam/agent-inspect/issues/14) |
| **3E** | Docs, CHANGELOG draft, release readiness | No publish until train gate passes |

Chunks may merge to `main` individually; **one** npm publish when the full train is coherent.

---

## Product boundaries (unchanged)

- Local-first, CLI-first, TypeScript-first
- No vendor upload by default
- No universal monkey-patching
- Instrumentation must not throw into user code
- `schemaVersion: "0.1"` manual traces remain readable

## Validation gates (each chunk)

```bash
pnpm build
pnpm typecheck
pnpm test
pnpm fixtures:check    # if fixtures touched
pnpm recipes:check     # if recipes touched
pnpm compat:smoke      # if exports or bundle layout change
pnpm test:all
```

## Release gate (before publish instruction)

```bash
pnpm install --frozen-lockfile
pnpm build
pnpm typecheck
pnpm test
pnpm test:all
pnpm fixtures:check
pnpm recipes:check
pnpm pack:smoke
pnpm compat:smoke
npm pack --dry-run
```

Plus: README / ROADMAP / CHANGELOG aligned; changeset added; maintainer explicit publish approval.

## Post–v1.2.0 foundation (available now)

From v1.2.0, consumers may use (experimental foundation):

- `PersistedInspectEvent`, `isPersistedInspectEvent`
- `traceEventToPersistedInspectEvent`, `inspectEventToPersistedInspectEvent`, etc.
- `persistedInspectEventsToRunTrees`, `traceEventsToPersistedRunTrees`

Storage and CLI still operate on v0.1 trace files by default.
