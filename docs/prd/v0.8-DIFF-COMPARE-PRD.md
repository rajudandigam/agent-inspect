## Goal


Help developers debug eval flakiness and regressions by comparing two agent runs and finding the first divergent step.


---


## User Story


> As a developer running agent evals, I want to compare a passing run with a failing run to quickly identify where execution diverged, so I can debug flaky tests and regressions faster.


---


## Scope


### In Scope
- Compare two runs by runId
- Find first divergent step
- Compare tree structure (same steps taken?)
- Compare inputs/outputs (same results?)
- Compare timing (significant differences?)
- Side-by-side display
- JSON output for scripting
- Ignore timing flag (for determinism checks)


### Out of Scope
- Replay/fork with execution (viewing only)
- Automatic test reruns
- Cassette recording/playback
- Time-travel debugging
- Multi-run comparison (>2 runs)
- Statistical analysis across many runs


## Comparison Dimensions


### 1. Structure Comparison


**Check**: Did both runs take the same steps?


```typescript
interface StructureDiff {
 same: boolean;
 runAOnly: string[];      // Steps only in run A
 runBOnly: string[];      // Steps only in run B
 common: string[];        // Steps in both
 firstDivergence?: number;
}
```


**Example**:
```text
Structure: ✖ Different steps taken


Run A: 5 steps
Run B: 3 steps


Steps only in Run A:
 - step 4: retry-search
 - step 5: finalize


Diverged after step 3 (tool:searchHotels)
```


### 2. Output Comparison


**Check**: Did the same steps produce the same outputs?


```typescript
interface OutputDiff {
 same: boolean;
 differences: Array<{
   step: string;
   runA: unknown;
   runB: unknown;
   diffType: "missing" | "extra" | "different" | "type-mismatch";
 }>;
}
```


**Example**:
```text
Outputs: ✖ Different


Step: tool:searchHotels
 Run A: ["Hotel A", "Hotel B"]  (2 items)
 Run B: []                       (0 items) ← DIFFERENT
 Diff type: missing data
```


### 3. Timing Comparison


**Check**: Were there significant timing differences?


```typescript
interface TimingDiff {
 significant: boolean;  // >20% difference
 steps: Array<{
   name: string;
   runADuration: number;
   runBDuration: number;
   percentDiff: number;
 }>;
}
```


**Example**:
```text
Timing: ⚠ Significant differences


Step: tool:searchHotels
 Run A: 2.31s
 Run B: 0.45s  (87% faster)
 Step: llm:generateMessage
 Run A: 1.82s
 Run B: 4.56s  (150% slower)
```


### 4. Error Comparison


**Check**: Did errors occur in different places?


```typescript
interface ErrorDiff {
 runAErrors: string[];
 runBErrors: string[];
 commonErrors: string[];
}
```

## Technical Design


### Core Algorithm


```typescript
function compareRuns(runA: InspectRunTree, runB: InspectRunTree): RunComparison {
 // 1. Flatten both trees to lists
 const flatA = flattenTree(runA);
 const flatB = flattenTree(runB);
  // 2. Align by step name (not index)
 const aligned = alignSteps(flatA, flatB);
  // 3. Find first divergence
 let firstDivergence: StepDiff | null = null;
  for (const [stepA, stepB] of aligned) {
   if (!stepB) {
     firstDivergence = {
       type: "missing-in-b",
       step: stepA.event.name,
       // ...
     };
     break;
   }
  
   if (!stepA) {
     firstDivergence = {
       type: "missing-in-a",
       step: stepB.event.name,
       // ...
     };
     break;
   }
  
   // Compare outputs
   if (!deepEqual(stepA.event.attributes?.output, stepB.event.attributes?.output)) {
     firstDivergence = {
       type: "output-diff",
       step: stepA.event.name,
       runA: stepA.event.attributes?.output,
       runB: stepB.event.attributes?.output,
     };
     break;
   }
  
   // Compare timing (if not ignored)
   if (!options.ignoreDuration && significantTimingDiff(stepA, stepB)) {
     firstDivergence = {
       type: "timing-diff",
       step: stepA.event.name,
       // ...
     };
     // Continue looking for output diffs (timing is lower priority)
   }
 }
  return {
   structure: compareStructure(flatA, flatB),
   outputs: compareOutputs(flatA, flatB),
   timing: compareTiming(flatA, flatB),
   errors: compareErrors(flatA, flatB),
   firstDivergence,
 };
}
```


### Step Alignment


**Challenge**: Steps may have different indices if one run skipped steps


**Solution**: Align by step name + order, not index


```typescript
function alignSteps(
 stepsA: InspectNode[],
 stepsB: InspectNode[]
): Array<[InspectNode | null, InspectNode | null]> {
 const aligned: Array<[InspectNode | null, InspectNode | null]> = [];
  let i = 0, j = 0;
  while (i < stepsA.length || j < stepsB.length) {
   const stepA = stepsA[i];
   const stepB = stepsB[j];
  
   if (!stepA) {
     aligned.push([null, stepB]);
     j++;
   } else if (!stepB) {
     aligned.push([stepA, null]);
     i++;
   } else if (stepA.event.name === stepB.event.name) {
     aligned.push([stepA, stepB]);
     i++;
     j++;
   } else {
     // Names don't match, determine which run is ahead
     if (stepsB.slice(j).find(s => s.event.name === stepA.event.name)) {
       // stepA exists later in B, so B is ahead
       aligned.push([null, stepB]);
       j++;
     } else {
       // stepA doesn't exist in B
       aligned.push([stepA, null]);
       i++;
     }
   }
 }
  return aligned;
}
```

## Use Cases


### 1. Eval Flakiness Debugging


**Scenario**: Test passes 90% of time, fails 10%


**Workflow**:
```bash
# Run test 10 times, save traces
for i in {1..10}; do
 npm test -- --grep "hotel booking" --trace
done


# Find a passing and failing run
agent-inspect list --status success | head -1  # run_pass123
agent-inspect list --status error | head -1    # run_fail456


# Compare
agent-inspect diff run_pass123 run_fail456
```


**Goal**: Identify non-deterministic step (e.g., tool returns empty array sometimes)


### 2. Regression Detection


**Scenario**: After code change, eval starts failing


**Workflow**:
```bash
# Before change
git checkout main
npm test -- --grep "hotel booking" --trace  # run_before789


# After change
git checkout feature-branch
npm test -- --grep "hotel booking" --trace  # run_after012


# Compare
agent-inspect diff run_before789 run_after012
```


**Goal**: Identify which step changed behavior


### 3. Performance Regression


**Scenario**: Same results but much slower


**Workflow**:
```bash
agent-inspect diff run_fast run_slow --focus timing
```


**Goal**: Identify which step became slower

## Go/No-Go Gate


**GO IF**:
- Eval flakiness is real pain
- Diff helps identify root cause faster
- 1+ team requests this feature


**NO-GO IF**:
- Evals are deterministic (no flakiness)
- Manual comparison sufficient
- No demand


**If NO-GO**: Skip diff, focus on core viewing


---


## Known Limitations


- Only compares 2 runs (not N runs)
- Step alignment heuristic may misalign complex divergences
- Deep object comparison may be slow for large outputs
- Timing differences affected by system load


---
