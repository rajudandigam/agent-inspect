## Standards Alignment


### OpenInference Span Schema


Reference: [OpenInference Specification](https://github.com/Arize-ai/openinference)


**Span Kinds**:
```typescript
type OpenInferenceSpanKind =
 | "LLM"
 | "AGENT"
 | "CHAIN"
 | "TOOL"
 | "RETRIEVER"
 | "RERANKER"
 | "EMBEDDING"
 | "GUARDRAIL"
 | "EVALUATOR"
 | "PROMPT";
```


**Required Attributes**:
- `openinference.span.kind`
- `input.value` (JSON string)
- `output.value` (JSON string)


**LLM-specific**:
- `llm.model_name`
- `llm.invocation_parameters` (JSON string)
- `llm.input_messages` (array of message objects)
- `llm.output_messages` (array of message objects)
- `llm.token_count.prompt`
- `llm.token_count.completion`
- `llm.token_count.total`


**Tool-specific**:
- `tool.name`
- `tool.description`
- `tool.parameters` (JSON schema)


### OpenTelemetry GenAI Conventions


Reference: [OTel GenAI Semantic Conventions](https://opentelemetry.io/docs/specs/semconv/gen-ai/)


**Required Attributes**:
- `gen_ai.system` (e.g., "openai", "anthropic", "google_generativeai")
- `gen_ai.request.model`
- `gen_ai.operation.name` (e.g., "chat", "completion")


**Optional Attributes**:
- `gen_ai.request.temperature`
- `gen_ai.request.max_tokens`
- `gen_ai.usage.input_tokens`
- `gen_ai.usage.output_tokens`
- `gen_ai.usage.total_tokens`
- `gen_ai.response.id`
- `gen_ai.response.finish_reasons` (array)

## Mapping: InspectEvent → OpenInference


```typescript
function toOpenInference(event: InspectEvent): OpenInferenceSpan {
 return {
   name: event.name,
   span_id: event.eventId,
   parent_id: event.parentId,
   start_time_unix_nano: event.timestamp * 1_000_000,
   end_time_unix_nano: event.endedAt
     ? event.endedAt * 1_000_000
     : event.timestamp * 1_000_000,
   status_code: mapStatus(event.status),
   status_message: event.status === "error" ? "Error" : "",
   attributes: {
     "openinference.span.kind": mapKind(event.kind),
     "input.value": JSON.stringify(event.attributes?.input),
     "output.value": JSON.stringify(event.attributes?.output),
    
     // LLM attributes (if LLM kind)
     ...(event.kind === "LLM" && {
       "llm.model_name": event.attributes?.model,
       "llm.token_count.prompt": event.attributes?.tokens?.input,
       "llm.token_count.completion": event.attributes?.tokens?.output,
       "llm.token_count.total":
         (event.attributes?.tokens?.input || 0) +
         (event.attributes?.tokens?.output || 0),
     }),
    
     // Tool attributes (if TOOL kind)
     ...(event.kind === "TOOL" && {
       "tool.name": event.name,
     }),
   },
 };
}


function mapKind(kind: InspectKind): OpenInferenceSpanKind {
 const mapping: Record<InspectKind, OpenInferenceSpanKind> = {
   LLM: "LLM",
   AGENT: "AGENT",
   CHAIN: "CHAIN",
   TOOL: "TOOL",
   RETRIEVER: "RETRIEVER",
   RUN: "CHAIN",  // Map RUN to CHAIN
   DECISION: "AGENT",
   RESULT: "CHAIN",
   ERROR: "CHAIN",
   LOGIC: "CHAIN",
   LOG: "CHAIN",
 };
  return mapping[kind] || "CHAIN";
}


function mapStatus(status?: string): "OK" | "ERROR" | "UNSET" {
 if (status === "ok" || status === "success") return "OK";
 if (status === "error") return "ERROR";
 return "UNSET";
}
```

## Mapping: InspectEvent → OTel GenAI


```typescript
function toOTelGenAI(event: InspectEvent): OTelSpan {
 return {
   traceId: generateTraceId(event.runId),
   spanId: event.eventId,
   parentSpanId: event.parentId,
   name: event.name,
   kind: "INTERNAL",
   startTimeUnixNano: event.timestamp * 1_000_000,
   endTimeUnixNano: event.endedAt
     ? event.endedAt * 1_000_000
     : event.timestamp * 1_000_000,
   status: { code: mapStatus(event.status) },
   attributes: [
     // OTel GenAI attributes
     ...(event.kind === "LLM" && [
       { key: "gen_ai.system", value: { stringValue: extractSystem(event) } },
       { key: "gen_ai.request.model", value: { stringValue: event.attributes?.model } },
       { key: "gen_ai.operation.name", value: { stringValue: "chat" } },
       { key: "gen_ai.usage.input_tokens", value: { intValue: event.attributes?.tokens?.input } },
       { key: "gen_ai.usage.output_tokens", value: { intValue: event.attributes?.tokens?.output } },
     ]),
    
     // OpenInference attributes (dual encoding)
     { key: "openinference.span.kind", value: { stringValue: mapKind(event.kind) } },
   ],
 };
}


function extractSystem(event: InspectEvent): string {
 const model = event.attributes?.model || "";
  if (model.includes("gpt")) return "openai";
 if (model.includes("claude")) return "anthropic";
 if (model.includes("gemini")) return "google_generativeai";
  return "unknown";
}
```

## Documentation Requirements


### Each Format Must Include


1. **Export command**
2. **Example output** (annotated)
3. **Import guide** for target platform
4. **Tested against** (platform + version)
5. **Known limitations**


**Example**:
```markdown
## OpenInference Export


Tested against:
- Phoenix 4.2.0 ✅ (verified)


Compatibility (untested, may work):
- Langfuse (supports OpenInference format)
- Braintrust SDK (supports OpenInference format)


Limitations:
- Nested chains >10 levels may not render correctly in Phoenix
- Token counts require explicit capture (not inferred)
- Only Phoenix compatibility is guaranteed in v0.7
```

## Known Limitations


- Export is read-only (no live streaming)
- Requires full trace file (no partial exports)
- Large traces (1000+ spans) may hit platform limits
- Custom attributes may not map perfectly
- Standards evolve (may need updates)
