# Extension registry submission template

**Labels:** `documentation`, `community contribution`, `adapter-sdk`, `good first issue`

**Difficulty:** Good first issue

**Milestone:** Adapter SDK Examples

## Problem

Community extensions need a **submission template** (package name, framework, privacy policy, conformance status, maintainer contact) for the extension registry docs.

## Proposed scope

- Add `docs/community/EXTENSION-SUBMISSION-TEMPLATE.md`.
- Link from adapter-sdk README and community contributing docs.

## Out of scope

- Automated registry hosting or npm org changes.

## Acceptance criteria

- [ ] Template fields complete and actionable
- [ ] Privacy/conformance sections included
- [ ] Docs-only PR
