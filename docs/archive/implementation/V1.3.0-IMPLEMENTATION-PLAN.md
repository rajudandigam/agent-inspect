## 2. Why this release exists

v1.2.0 added a **source-agnostic persisted-event foundation** (`PersistedInspectEvent`, converters, in-memory tree bridge) without changing default trace writing.

v1.3.0 makes traces easier to **group**, **share safely**, and **inspect during streaming workflows**:

- **Correlation metadata** links runs to requests, eval cases, CI jobs, and decision branches.
- **Redaction profiles** (chunk 2) tighten what leaves the machine on export.
- **LangChain streaming metadata** (chunk 3) records stream lifecycle without raw token capture by default.

Correlation supports future **CI artifacts**, **trace-to-eval**, **stats/cohort**, and multi-service debugging â€” all local-first.

---

## 3. Goals

- Add `correlationId` / `requestId` / `decisionId` / `groupId` foundation on `inspectRun` / `maybeInspectRun`.
- Add redaction profiles and share-safe export mode in **chunk 2**.
- Add LangChain streaming metadata-only support in **chunk 3**.
- Preserve **local-first / no-vendor-upload** boundary.
- Keep manual trace writing at **`schemaVersion: "0.1"`**.
- Keep **v0.2** as experimental foundation (converters only; not default write format).
