# V1.9.0 Execution Plan

## Train

```yaml
train: "v1.9.0"
theme: "Adoption leverage"
executionMode: "autonomous-release-train"
branch: "main"
releasePolicy: "minor-only; do not prepare patch releases"
```

## Goal

Convert existing AgentInspect capability into repeated local use by reducing integration friction, adding a safe explain workflow, promoting adapters, and preparing the v2 root API contract.

## Product boundaries

- Keep AgentInspect local-first, CLI-first, TypeScript-first, and dependency-light at root.
- Do not add hosted SaaS, dashboard, telemetry upload, provider pricing, default replay, raw chain-of-thought capture, or default network behavior.
- New packages may be developed as private workspace packages during the train. First public publication remains a manual maintainer gate.
- Do not change package versions, create changesets, tag, publish, or create a GitHub release during implementation chunks.
- v1.9.0 is the next release target. Do not prepare or publish a v1.8.1 patch release.

## Train stop conditions

Stop autonomous execution on unrelated worktree changes, material plan drift, public breaking changes, schema changes, new root/core dependencies, unapproved network behavior, validation that cannot be repaired in the current scope, missing credentials, partial publication, or any release operation requiring maintainer authorization.
