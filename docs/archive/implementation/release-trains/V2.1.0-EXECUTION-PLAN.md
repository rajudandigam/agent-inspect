# v2.1.0 Execution Plan

## Train

```yaml
train: "v2.1.0"
theme: "Utility triangle: eval and redact"
baselineVersion: "2.0.0"
executionMode: "autonomous-release-train"
```

## Goal

Turn AgentInspect from a trace utility into the local TypeScript agent toolkit:

```text
trace + eval + redact
```

v2.1 should ship two optional utilities:

- `@agent-inspect/eval`
- `@agent-inspect/redact`

The root package remains dependency-light and local-first.

## Product boundaries

AgentInspect remains:

- local-first;
- CLI-first;
- TypeScript-first;
- deterministic by default;
- safe by default;
- explicit about network behavior.

v2.1 must not add:

- hosted service;
- default network calls;
- LLM judge by default;
- provider pricing;
- prompt registry;
- dataset platform;
- raw chain-of-thought capture;
- replay/cassette behavior;
- root runtime dependency bloat.

## Release strategy

Use small commits but one release train.

Do not publish per chunk.

v2.1 is publishable only after:

- all public docs align;
- `@agent-inspect/eval` and `@agent-inspect/redact` package boundaries are clear;
- full release readiness passes;
- package smoke covers every public package;
- the maintainer explicitly authorizes versioning and publication.
