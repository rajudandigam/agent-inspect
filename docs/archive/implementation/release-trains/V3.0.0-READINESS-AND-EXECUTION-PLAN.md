# v3.0.0 Readiness and Execution Plan

## Status

v3.0 is conditional. It is not guaranteed because v2.x is implemented.

Do not begin v3 implementation unless v3 readiness gates pass.

## v3 goal

Turn AgentInspect from a package family into a stable local trace-tooling ecosystem.

## v3 readiness gates

Proceed only when all are true or explicitly waived by the maintainer with evidence:

- v2 demonstrates retained usage;
- several external projects emit AgentInspect-compatible traces;
- reporter/check workflows recur;
- at least one third-party adapter is maintained outside the core repo;
- users ask for extensibility rather than basic setup;
- local-first remains a meaningful product advantage.

If these signals do not appear, narrow the product to the best-adopted adapter/workflow.

## v3 non-goals

- hosted SaaS;
- billing;
- prompt management;
- dataset management;
- production alerting platform;
- automatic production replay;
- universal auto-remediation;
- full APM replacement;
- default network upload;
- root provider SDK dependencies.

## v3 stable extension contracts

Candidate contracts:

```ts
interface TraceSource {}
interface TraceReader {}
interface TraceWriter {}
interface TraceTransform {}
interface TraceCheck {}
interface TraceRenderer {}
interface TraceIndexer {}
interface TraceAdapter {}
```

Also stabilize:

- semantic extension fields;
- adapter registration;
- conformance runner;
- compatibility badge;
- versioning policy;
- migration policy.

## v3 architecture

Every surface consumes the same stable schema:

- CLI;
- CI reporters;
- static reports;
- local viewer;
- read-only MCP server;
- IDE integration;
- external adapter tools;
- diagnostic runtime.

Storage model:

- JSONL remains the portable durable artifact;
- optional indexes are derived;
- indexes are rebuildable;
- no hosted database required;
- no proprietary cloud protocol mandatory.

Goal:
Verify v3 should happen.

In scope:

- external adapter evidence;
- issue/discussion demand;
- package usage trends;
- extension-point requests.

Out of scope:
- code changes.


### Chunk 2 — adapter SDK package

Goal:
Create optional adapter SDK using existing conformance utilities.

In scope:
- mapping helpers;
- fixture generators;
- capture/privacy checklist;
- adapter registration;
- conformance runner integration.

Out of scope:
- vendor-specific adapters not demanded.

Suggested commit:

```text

```

### Chunk 3 — transform and renderer contracts

Goal:
Support third-party transforms/renderers while keeping schema stable.

In scope:
- transform pipeline;
- renderer interface;
- safety hooks;
- evidence and warnings;
- docs/examples.

Out of scope:
- arbitrary plugin execution in root CLI without safety boundaries.

Suggested commit:

```text
feat: add trace transform and renderer contracts
```

Goal:

Out of scope:

## v3 validation gate

```bash
pnpm install --frozen-lockfile
pnpm build
pnpm typecheck
pnpm test
pnpm test:coverage
pnpm fixtures:check
pnpm recipes:check
pnpm size
pnpm test:all
pnpm pack:smoke
pnpm compat:smoke
npm pack --dry-run
git diff --check
```

Also run:
- extension SDK package tests;
- conformance runner against official adapters;
- clean install tests for any new public packages;
- viewer/MCP/IDE smoke where applicable.

## Stop conditions

Stop v3 if:
- adoption gates fail;
- users mostly need better docs rather than extensions;
- extension contracts force breaking schema changes not justified by demand;
- implementation drifts toward SaaS/dashboard/marketplace/APM.
