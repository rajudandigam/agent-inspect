# v3.1.0 Execution Plan

## Status

Active post-v3 adoption train. **Goal:** reduce first-use and real-project integration friction.

**Product thesis:** Install → one command → one trace → one check → one safe artifact in under five minutes.

## Non-goals

Hosted dashboard, SaaS, marketplace, dependency auto-install without confirmation, telemetry, network upload, framework deps in root/core.

## Execution chunks

### Chunk 0 — audit current package state and publish verification

Goal: Verify v3.0 publication and repo state before implementation.

In scope: npm version checks, harness audit, state file reset.

Out of scope: runtime changes.

Suggested commit: `docs: start post-v3 adoption roadmap`

### Chunk 1 — public `@agent-inspect/harness`

Goal: Publish harness or ship recipes-only if quality gate fails.

In scope: remove `private`, docs, package smoke, recipes, tests, boundary updates.

Out of scope: Nest-specific env, real model calls, root deps.

**Manual gate:** first npm publication + Trusted Publisher for `@agent-inspect/harness`.

Suggested commit: `feat(harness): make fixture runner package public`

### Chunk 2 — `agent-inspect init`

Goal: First-run initializer (config, trace dir, starter references, optional CI snippet).

CLI: `init`, `--framework`, `--ci github`, `--dry-run`, `--yes`, `--json`.

Out of scope: auto `npm install`, network, telemetry.

Suggested commit: `feat(cli): add init command`

### Chunk 3 — `agent-inspect doctor`

Goal: Local setup diagnostics with deterministic JSON output.

Checks: Node, version, CLI, trace dir, import smoke, env vars, optional packages, version mismatch.

Out of scope: install, upload, mutation.

Suggested commit: `feat(cli): add doctor command`

### Chunk 4 — starter templates

Goal: Deterministic starters under `examples/starters/` (custom, ai-sdk, openai-agents, langchain, ci-eval-redact, harness-nestjs).

Suggested commit: `docs: add adoption starter templates`

### Chunk 5 — README/docs onboarding update

Goal: Lead with init → observe → adapter → harness → doctor → check/eval/redact.

Suggested commit: `docs: make v3.1 onboarding path obvious`

### Chunk 6 — release readiness

Goal: Full release gate, changeset for v3.1.0, Version Packages PR, publish verification.

Suggested commit: `docs: prepare v3.1 release readiness`

## Validation gate

```bash
pnpm install --frozen-lockfile
pnpm build
pnpm typecheck
pnpm test
pnpm recipes:check
pnpm pack:smoke
pnpm compat:smoke
git diff --check
```
