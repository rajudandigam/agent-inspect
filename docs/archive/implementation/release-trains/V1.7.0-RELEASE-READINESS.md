# v1.7.0 Release Readiness

**Date:** 2026-06-25  
**Starting commit:** `ce92e2b` (`test: add adapter conformance fixture matrix`)  
**Status:** published 2026-06-26

## Scope

v1.7.0 builds on the v1.6 runtime/reader/writer foundation with framework-native adapter adoption:

- experimental `@agent-inspect/ai-sdk` package for Vercel AI SDK v6 telemetry integrations;
- metadata-only AI SDK run, step, tool, error, streaming, and token-usage mapping;
- local no-network AI SDK recipe coverage;
- OpenAI Agents JS tracing processor RFC and scaffold-only package boundary;
- LangGraph support decision through `@agent-inspect/langchain`;
- adapter conformance matrix for local-first defaults and fixture expectations.

## Package And Release Strategy

The repository publish workflow runs `changesets/action@v1` with `publish: pnpm run release`. On pushes to `main`, the workflow first runs release-train gates, then either creates a Version Packages PR when changesets exist or publishes when versioned package files are merged.

Release preparation:

- Added a Changeset for a linked minor release of `agent-inspect`, `@agent-inspect/langchain`, `@agent-inspect/tui`, and `@agent-inspect/ai-sdk`.
- Made `@agent-inspect/ai-sdk` publishable with `publishConfig.access: "public"`.
- Kept `@agent-inspect/openai-agents` private, scaffold-only, and ignored by Changesets because runtime mapping is not implemented.
- Merged the Version Packages PR, which bumped the public packages to `1.7.0`.

Release outcome:

## Compatibility

- Stable root imports remain unchanged.
- Manual traces still write `schemaVersion: "0.1"`.
- v0.1 and v0.2 AgentInspect JSONL traces remain readable.
- v0.2 adapter events remain experimental and additive.
- Root/core packages do not depend on AI SDK, OpenAI Agents, LangGraph, OpenTelemetry, or LangChain.
- Node support remains `>=20`.

## Security And Network

- No default telemetry upload, vendor sink, hosted ingestion, collector client, or replay behavior is added.
- AI SDK examples require `recordInputs: false` and `recordOutputs: false`.
- AI SDK capture defaults remain metadata-only; prompts, messages, generated text, stream chunks, tool inputs/outputs, headers, request bodies, and response bodies are not persisted by default.
- OpenAI Agents scaffold does not install itself globally, perform network I/O, or preserve default backend export behavior by default.
- LangGraph support remains through explicit LangChain callback boundaries unless future fixtures prove a gap.
