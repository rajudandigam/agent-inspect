## Decision

**Proceed with a narrow v3.0 train** focused on **documented extension contracts and optional adapter SDK** — not a hosted product expansion.

Rationale:

1. v2.x delivered stable schema, checks, eval, sessions, guardrails/circuit, and optional viewer/MCP surfaces.
2. Adapter conformance infrastructure exists (`@agent-inspect/harness`, conformance runner) — natural foundation for v3 SDK.
3. Demand for **third-party adapter authoring** is plausible from design-partner targets even without public recurrence metrics yet.
4. Stop conditions remain active: no SaaS, marketplace, APM, or schema breaks without evidence.

**Deferred if gates worsen:** community registry marketplace, IDE extension, hosted indexer, plugin execution in root CLI without safety boundaries.

---

## v3 scope lock (this train)

**In:**

- Extension contract RFC (chunk 1)
- Optional `@agent-inspect/adapter-sdk` (chunk 2)
- Transform/renderer contracts (chunk 3)
- Optional indexer contract (chunk 4)

**Out:**

- Automatic package scanning marketplace
- Breaking schema changes without migration path
- Production replay or auto-remediation
