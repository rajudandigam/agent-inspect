# v1.6.0 Execution Plan

## Status

**Published baseline:** `1.5.0`  
**Train:** `v1.6.0`  
**Goal:** instance-scoped local tracing plus universal local trace ingestion  
**Release policy:** small manually reviewed commits; one minor release after full readiness

This plan reflects work already completed on `main` and inserts a foundation-hardening checkpoint before external readers.

## Non-negotiable compatibility

- Global/manual APIs continue working.
- Global/manual writes remain `schemaVersion: "0.1"` during 1.x.
- v0.1 and v0.2 remain readable.
- No destructive migration or third persisted model.
- No default network behavior.
- No root/core framework or OTel SDK dependencies.
- No provider pricing/token-counting engine.
- New APIs remain experimental.
- New specialized APIs prefer `/writers`, `/readers`, or `/advanced`.
- ESM, CJS, TypeScript declarations, package smoke, and existing CLI behavior remain valid.

## Hardening checkpoint

### H1 — Persisted-event safety and writer no-throw contract

Goal: make the v0.2 persistence path at least as safe as the v0.1 path.

Required:

- central `preparePersistedInspectEventForWrite`;
- metadata/error/input/output/attribute bounds as supported by the schema;
- final serialized-size enforcement;
- safe cloning/serialization;
- invalid/uncloneable event isolation;
- no synchronous instrumentation exceptions;
- built-in writer consistency;
- focused security and failure tests.

Out of scope:

- schema changes;
- network writers;
- capture of full prompts/outputs;
- external readers.

Likely files:

- `packages/core/src/trace-event-safety.ts`
- `packages/core/src/inspector.ts`
- `packages/core/src/writers/index.ts`
- `packages/core/test/inspector.test.ts`
- `packages/core/test/writers/index.test.ts`
- focused safety tests

Suggested commit:

```text
fix: harden persisted event writes
```

### H2 — Experimental API boundary cleanup

Goal: avoid publishing misleading options or unnecessary root surface.

Decide and implement:

- remove or implement `capture`;
- remove or clearly define `traceDir` and `silent` on `createInspector`;
- prefer writer-owned output configuration;
- expose diagnostics without requiring full public runtime access;
- move low-level runtime symbols to `/advanced`;
- keep writers/readers on their subpaths;
- minimize new root exports while preserving already published APIs;
- update API/export/smoke tests and docs.

Suggested commit:

```text
refactor: tighten experimental runtime API boundaries
```

### H3 — Reader fidelity, resolved input, and detection policy

Goal: establish a scalable reader foundation before adding external formats.

Required:

- resolve file/buffer/string/stdin input once;
- preserve native v0.2 rows in mixed inputs;
- convert only v0.1 rows;
- retain source identity, confidence, trace/span context, unknown attributes, and ordering;
- define deterministic reader priority;
- define minimum confidence and close-candidate ambiguity behavior;
- apply input-size limits;
- attach source-file/line warnings where possible;
- avoid repeated full file reads.

Suggested commit:

```text
refactor: harden trace reader input and fidelity
```
