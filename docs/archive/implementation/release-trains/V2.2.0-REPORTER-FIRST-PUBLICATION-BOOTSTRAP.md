# v2.2.0 Reporter First-Publication Bootstrap

**Date:** 2026-06-28
**Purpose:** create npm package records for `@agent-inspect/vitest` and `@agent-inspect/jest` so Trusted Publishing can be enabled before the normal v2.2 release workflow.

## State

The reporter packages are prepared as public bootstrap packages at `2.1.0`, matching the current published baseline. They remain ignored by Changesets until the maintainer completes manual bootstrap publication and enables Trusted Publishing.

Do not use this document to publish v2.2.0. The normal v2.2 release prep remains a separate maintainer-authorized step after package records and Trusted Publishing are ready.

## Manual Bootstrap Commands

From a clean checkout of `main` at the bootstrap-prep commit:

```bash
git status --short
git pull --ff-only origin main
pnpm install --frozen-lockfile
CI=true pnpm build
CI=true pnpm pack:smoke
```

Confirm npm still does not have the package records:

```bash
npm view @agent-inspect/vitest version dist-tags --json
npm view @agent-inspect/jest version dist-tags --json
```

Expected before bootstrap: both commands return npm `E404`.

Pack the two reporter packages locally and inspect their packed manifests:

```bash
mkdir -p /tmp/agent-inspect-reporter-bootstrap
pnpm --dir packages/vitest pack --pack-destination /tmp/agent-inspect-reporter-bootstrap
pnpm --dir packages/jest pack --pack-destination /tmp/agent-inspect-reporter-bootstrap
tar -xOf /tmp/agent-inspect-reporter-bootstrap/agent-inspect-vitest-2.1.0.tgz package/package.json
tar -xOf /tmp/agent-inspect-reporter-bootstrap/agent-inspect-jest-2.1.0.tgz package/package.json
```

Check the packed manifests before publishing:

- `private` must be absent.
- `version` must be `2.1.0`.
- `publishConfig.access` must be `public`.
- `dependencies.agent-inspect` must be `2.1.0`, not `workspace:*`.
- `@agent-inspect/vitest` must have peer dependency `vitest: ^2.1.0`.
- `@agent-inspect/jest` must have peer dependency `jest: ^29.0.0 || ^30.0.0`.

Publish the bootstrap records manually:

```bash
pnpm --dir packages/vitest publish --access public --tag bootstrap --no-git-checks
pnpm --dir packages/jest publish --access public --tag bootstrap --no-git-checks
```

Verify npm registry state:

```bash
npm view @agent-inspect/vitest version dist-tags --json
npm view @agent-inspect/jest version dist-tags --json
```

After both package records exist, enable Trusted Publishing for:

```text
@agent-inspect/vitest
@agent-inspect/jest
```

Then report the npm `view` output back into the release train. The next automated step is v2.2 release prep: remove the reporter packages from Changesets ignore, add them to the linked public package set, create the linked minor changeset, validate, push, and let the normal Version Packages PR / publish workflow run.

## Stop Conditions

Stop and ask for confirmation if:

- `pnpm pack:smoke` fails;
- either packed manifest still contains `workspace:*`;
- either publish command publishes with `latest` unexpectedly and you want to avoid that tag before v2.2;
- npm returns an ownership, 2FA, provenance, package-name, or access error;
- Trusted Publishing cannot be enabled after the bootstrap package records exist.
